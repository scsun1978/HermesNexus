# Plan

Use this command before any multi-step change.

## Instructions

1. Read the MVP documents.
2. Inspect the repo state.
3. List the exact files likely to change.
4. Break the work into small implementation steps.
5. Include verification for each step.
6. Keep the plan aligned with MVP Phase 1 only.

## Planning Constraints

- Do not invent extra scope.
- Keep the plan compatible with the development/test server: `scsun@172.16.100.101:22`.
- Prefer the smallest reversible increments.
- Separate local development from deployment validation.

## Plan Format

- Goal
- Assumptions
- Files to edit
- Step-by-step tasks
- Verification commands
- Risks / out of scope
