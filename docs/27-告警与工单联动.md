# 27 告警与工单联动

## 目标

让告警不只是“通知”，而是能驱动 Hermes 生成分析、建议和自动化处置。

## 联动链路

1. 监控系统触发告警
2. 云端接收事件
3. Hermes 进行初步分析
4. 生成处置建议或自动任务
5. 必要时创建工单
6. 执行结果写回告警与工单系统

## 告警分级

### 低级
- 仅记录
- 进入观察队列

### 中级
- 通知运维
- 建议检查
- 可选自动巡检

### 高级
- 自动建任务
- 需要审批
- 可触发回滚或应急动作

## Hermes 的作用

Hermes 可以：
- 汇总上下文
- 找到相似历史事件
- 生成分析摘要
- 推荐处置步骤
- 自动调用技能执行部分动作

## 工单联动字段

- alert_id
- incident_id
- region_id
- device_id
- severity
- summary
- suggested_action
- approval_state
- resolution_state

## 设计原则

- 告警必须能追溯到任务和设备
- 每个处置建议都要有依据
- 自动化动作必须保留人工兜底
- 工单不是替代告警，而是承接告警后的治理流程
