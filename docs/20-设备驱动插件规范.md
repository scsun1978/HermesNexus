# 20 设备驱动插件规范

## 目标

把不同设备厂商、协议和现场差异封装成独立插件，避免主系统被设备细节污染。

## 插件职责

- 连接设备
- 认证
- 读取状态
- 下发命令
- 处理响应
- 标准化错误

## 插件接口建议

### 必备方法
- is_available
- connect
- read_status
- execute_action
- verify_result
- rollback
- disconnect

### 元数据
- name
- vendor
- device_type
- transport
- capabilities
- risk_level

## 插件生命周期

1. 加载
2. 初始化
3. 探测能力
4. 执行动作
5. 验证结果
6. 清理连接

## 插件设计原则

- 协议与业务分离
- 厂商差异留在插件里
- 输出统一成结构化结果
- 错误必须规范化
- 不要把临时脚本写成长期插件

## 推荐输出格式

```json
{
  "success": true,
  "device_id": "device-1",
  "action": "read_status",
  "result": {
    "status": "ok"
  },
  "warnings": []
}
```

## Hermes 适配方式

Hermes skill 负责编排，插件负责硬件和协议细节。
