# HermesNexus v1.2 批量操作 API 文档

Date: 2026-04-15  
Version: 1.2.0  
Status: 已实现

## 概述

本文档描述 HermesNexus v1.2 中实现的批量操作 API，包括资产批量操作和任务批量操作的完整接口定义。

## 统一响应结构

所有批量操作返回统一的 `BatchOperationResponse` 结构：

```json
{
  "operation_id": "batch-op-uuid",
  "operation_type": "asset_create | asset_update | asset_delete | asset_deactivate | task_create | task_dispatch",
  "status": "pending | running | completed | partial_success | failed",
  "summary": {
    "total_items": 10,
    "successful_items": 8,
    "failed_items": 2,
    "skipped_items": 0,
    "success_rate": 80.0,
    "operation_id": "batch-op-uuid"
  },
  "results": [
    {
      "id": "item-001",
      "success": true,
      "message": "操作成功",
      "error_code": null,
      "data": {},
      "created_at": "2026-04-15T10:30:00Z"
    }
  ],
  "started_at": "2026-04-15T10:30:00Z",
  "completed_at": "2026-04-15T10:30:05Z",
  "error_summary": {
    "validation_error": 1,
    "duplicate_error": 1
  }
}
```

### 状态说明

- `pending`: 批量操作已创建，等待执行
- `running`: 批量操作正在执行中
- `completed`: 所有项目均成功执行
- `partial_success`: 部分项目成功，部分失败
- `failed`: 批量操作失败（如验证失败、系统错误）

## 资产批量操作 API

### 1. 批量创建资产

**端点**: `POST /api/v1/batch/assets/create`

**描述**: 批量创建多个资产，支持幂等性和部分失败处理。

**请求体**:
```json
{
  "assets": [
    {
      "asset_id": "asset-001",
      "name": "Server 1",
      "asset_type": "linux_host",
      "metadata": {
        "ip_address": "192.168.1.1",
        "ssh_port": 22
      }
    },
    {
      "asset_id": "asset-002",
      "name": "Server 2",
      "asset_type": "linux_host",
      "metadata": {
        "ip_address": "192.168.1.2"
      }
    }
  ],
  "stop_on_first_error": false,
  "validate_only": false,
  "idempotency_key": "create-assets-2026-04-15-001"
}
```

**字段说明**:
- `assets`: 资产列表，最多 100 个
- `stop_on_first_error`: 遇到错误是否停止处理（默认 false）
- `validate_only`: 仅验证数据不执行创建（默认 false）
- `idempotency_key`: 幂等性键，相同 key 返回缓存结果

**验证规则**:
- 必需字段: `asset_id`, `name`, `asset_type`
- 有效资产类型: `linux_host`, `windows_host`, `network_device`, `iot_device`
- `asset_id` 不能为空且必须唯一

**响应**: `BatchOperationResponse`

### 2. 批量更新资产

**端点**: `POST /api/v1/batch/assets/update`

**描述**: 批量更新多个资产的指定字段。

**请求体**:
```json
{
  "asset_ids": ["asset-001", "asset-002"],
  "updates": {
    "status": "active",
    "metadata": {
      "environment": "production",
      "updated_by": "admin"
    }
  },
  "stop_on_first_error": false,
  "validate_only": false,
  "idempotency_key": "update-assets-2026-04-15-001"
}
```

**字段说明**:
- `asset_ids`: 要更新的资产 ID 列表，最多 100 个
- `updates`: 要更新的字段和值
- 其他字段同批量创建

**响应**: `BatchOperationResponse`

### 3. 批量删除资产

**端点**: `POST /api/v1/batch/assets/delete`

**描述**: 批量删除多个资产。

**请求体**:
```json
{
  "asset_ids": ["asset-001", "asset-002"],
  "force": false,
  "stop_on_first_error": false,
  "idempotency_key": "delete-assets-2026-04-15-001"
}
```

**字段说明**:
- `asset_ids`: 要删除的资产 ID 列表，最多 100 个
- `force`: 强制删除（忽略关联检查）
- `stop_on_first_error`: 遇到错误是否停止
- `idempotency_key`: 幂等性键

**响应**: `BatchOperationResponse`

### 4. 批量停用资产

**端点**: `POST /api/v1/batch/assets/deactivate`

**描述**: 批量停用多个资产（软删除）。

**请求体**:
```json
{
  "asset_ids": ["asset-001", "asset-002"],
  "stop_on_first_error": false,
  "idempotency_key": "deactivate-assets-2026-04-15-001"
}
```

**字段说明**:
- `asset_ids`: 要停用的资产 ID 列表，最多 100 个
- `stop_on_first_error`: 遇到错误是否停止
- `idempotency_key`: 幂等性键

**响应**: `BatchOperationResponse`

## 任务批量操作 API

### 1. 批量创建任务

**端点**: `POST /api/v1/batch/tasks/create`

**描述**: 批量创建多个任务，支持并行执行。

**请求体**:
```json
{
  "tasks": [
    {
      "task_id": "task-001",
      "name": "Update System",
      "target_device_id": "asset-001",
      "command": "yum update -y",
      "timeout": 300
    },
    {
      "task_id": "task-002",
      "name": "Check Disk",
      "target_device_id": "asset-002",
      "command": "df -h",
      "timeout": 60
    }
  ],
  "stop_on_first_error": false,
  "validate_only": false,
  "idempotency_key": "create-tasks-2026-04-15-001",
  "parallel_execution": true,
  "max_parallel_tasks": 5
}
```

**字段说明**:
- `tasks`: 任务列表，最多 50 个
- `parallel_execution`: 是否并行执行（默认 true）
- `max_parallel_tasks`: 最大并行任务数（1-50，默认 10）
- 其他字段同资产批量创建

**验证规则**:
- 必需字段: `task_id`, `name`, `target_device_id`, `command`
- `task_id` 必须唯一
- `timeout` 范围: 10-3600 秒

**响应**: `BatchOperationResponse`

### 2. 批量下发任务

**端点**: `POST /api/v1/batch/tasks/dispatch`

**描述**: 将多个任务批量下发到指定节点执行。

**请求体**:
```json
{
  "task_ids": ["task-001", "task-002"],
  "target_node_ids": ["node-001", "node-002"],
  "dispatch_options": {
    "parallel": true,
    "timeout": 300,
    "priority": "normal"
  },
  "idempotency_key": "dispatch-tasks-2026-04-15-001"
}
```

**字段说明**:
- `task_ids`: 要下发的任务 ID 列表，最多 50 个
- `target_node_ids`: 目标节点 ID 列表
- `dispatch_options`: 下发选项
  - `parallel`: 是否并行下发
  - `timeout`: 下发超时时间
  - `priority`: 优先级（low, normal, high）

**响应**: `BatchOperationResponse`

## 错误码与分类

系统自动分类错误类型并在 `error_summary` 中统计：

| 错误码 | 说明 | 示例 |
|-------|------|------|
| `validation_error` | 数据验证失败 | 缺少必需字段、字段格式错误 |
| `duplicate_error` | 重复数据冲突 | ID 已存在、唯一索引冲突 |
| `not_found_error` | 资源不存在 | 资产 ID 不存在 |
| `timeout` | 操作超时 | 数据库查询超时、SSH 连接超时 |
| `permission_error` | 权限不足 | 无操作权限 |
| `connection_error` | 连接失败 | 数据库连接失败、网络不通 |
| `unknown_error` | 未知错误 | 其他未分类错误 |

## 幂等性保证

所有批量操作支持通过 `idempotency_key` 实现幂等性：

1. 首次请求：执行操作并缓存结果
2. 相同 key 的重复请求：直接返回缓存结果
3. 缓存键格式：`{operation_type}:{idempotency_key}`
4. 缓存有效期：当前会话期间

**幂等性响应示例**:
```json
{
  "operation_id": "original-operation-id",
  "operation_type": "asset_create",
  "status": "completed",
  "message": "操作已存在，返回缓存结果",
  "is_idempotent": true
}
```

## 部分失败处理

批量操作默认支持部分失败处理：

1. **continue_on_error**: 遇到错误继续处理后续项目
2. **error_threshold**: 失败阈值（默认 50%）
3. **save_partial_results**: 保存部分成功结果

**部分失败响应示例**:
```json
{
  "operation_id": "batch-op-partial",
  "operation_type": "asset_create",
  "status": "partial_success",
  "summary": {
    "total_items": 10,
    "successful_items": 7,
    "failed_items": 3,
    "success_rate": 70.0
  },
  "results": [
    {
      "id": "asset-001",
      "success": true,
      "message": "资产创建成功"
    },
    {
      "id": "asset-002",
      "success": false,
      "message": "缺少必需字段: name",
      "error_code": "validation_error"
    }
  ],
  "error_summary": {
    "validation_error": 2,
    "duplicate_error": 1
  }
}
```

## 性能限制

| 参数 | 限制 | 说明 |
|------|------|------|
| 单次批量大小 | 资产 100 个，任务 50 个 | 防止单次请求过大 |
| 并发任务数 | 1-50 个 | 默认 10 个 |
| 请求超时 | 默认 300 秒 | 可配置 |
| 幂等性缓存 | 当前会话 | 重启后清空 |

## 使用示例

### Python 客户端示例

```python
import requests
import json

# 批量创建资产
response = requests.post(
    "http://localhost:8000/api/v1/batch/assets/create",
    json={
        "assets": [
            {
                "asset_id": f"server-{i}",
                "name": f"Server {i}",
                "asset_type": "linux_host"
            }
            for i in range(1, 11)
        ],
        "idempotency_key": "batch-create-2026-04-15"
    }
)

result = response.json()
print(f"操作ID: {result['operation_id']}")
print(f"成功率: {result['summary']['success_rate']}%")
print(f"成功: {result['summary']['successful_items']}")
print(f"失败: {result['summary']['failed_items']}")
```

### cURL 示例

```bash
# 批量创建资产
curl -X POST "http://localhost:8000/api/v1/batch/assets/create" \
  -H "Content-Type: application/json" \
  -d '{
    "assets": [
      {"asset_id": "server-1", "name": "Server 1", "asset_type": "linux_host"},
      {"asset_id": "server-2", "name": "Server 2", "asset_type": "linux_host"}
    ],
    "idempotency_key": "batch-create-001"
  }'

# 批量下发任务
curl -X POST "http://localhost:8000/api/v1/batch/tasks/dispatch" \
  -H "Content-Type: application/json" \
  -d '{
    "task_ids": ["task-1", "task-2"],
    "target_node_ids": ["node-1"],
    "idempotency_key": "dispatch-001"
  }'
```
