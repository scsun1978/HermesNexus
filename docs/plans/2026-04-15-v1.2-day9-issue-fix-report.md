## 🚨 发现的阻塞问题

用户测试发现了两个实锤阻塞问题，Day 9 不能标记为完成：

### 问题1: Python类型注解错误 ❌
**错误位置**: `tests/e2e/conftest.py:47`
```python
# 错误的类型注解
def test_storage() -> Generator[AuditStorage, None]:
    pass

# Python 3.11下正确格式
def test_storage() -> Generator[AuditStorage, None, None]:
    pass
```

**影响范围**: 
- pytest无法在Python 3.11下加载conftest.py
- 导致所有E2E测试无法运行
- 声称的"11/11 smoke passed"在实际环境下无法复现

**修复状态**: ✅ 已修复

### 问题2: API路由劫持导致422错误 ❌
**错误位置**: `cloud/api/task_api.py:430-450` 和 `cloud/api/main.py:129`

**问题描述**:
- `/api/v1/jobs`被legacy路由劫持
- `jobs_router`的兼容端点使用`*args, **kwargs`
- FastAPI无法匹配参数，返回422错误
- 新的`list_jobs`和`create_job`端点无法正常工作

**错误响应**:
```json
{
  "detail": [{
    "loc": ["query", "args"],
    "msg": "field required",
    "type": "missing"
  }, {
    "loc": ["query", "kwargs"], 
    "msg": "field required",
    "type": "missing"
  }]
}
```

**影响范围**:
- 任务列表API无法使用
- 任务创建API无法使用
- FastAPI路由混乱

**修复状态**: ⚠️ 需要重构legacy路由逻辑
