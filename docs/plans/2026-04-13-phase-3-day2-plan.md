## 3. 核心概念设计

### 3.1 权限模型架构

```
权限维度:
├── 维度1: 用户身份（User/Node/Admin）
├── 维度2: 租户隔离（Tenant）
├── 维度3: 区域限制（Region/Zone）
├── 维度4: 设备类型（AssetType）
├── 维度5: 操作动作（Action）
└── 维度6: 风险等级（RiskLevel）

权限判断逻辑:
IF (用户权限匹配) AND (租户/区域允许) AND (设备类型允许) THEN
    IF (风险等级 == HIGH) THEN
        需要审批
    ELSE
        直接执行
    ENDIF
ELSE
    默认拒绝
ENDIF
```

### 3.2 风险分级定义

**低风险（LOW）**:
- 查询操作（GET）
- 只读任务
- 不影响系统状态
- 示例：查询设备状态、查看日志

**中风险（MEDIUM）**:
- 修改操作（PUT/PATCH）
- 非关键配置变更
- 可快速回滚的操作
- 示例：修改设备描述、调整任务优先级

**高风险（HIGH）**:
- 删除操作（DELETE）
- 关键配置变更
- 影响业务连续性的操作
- 示例：删除设备、重启服务、修改网络配置
