## 🚀 发布流程

### 1. 准备发布
```bash
# 1. 创建发布分支
git checkout -b release/vX.Y.Z

# 2. 更新版本号
# 更新各个version文件

# 3. 运行完整CI检查
bash scripts/ci-check.sh

# 4. 创建构建产物
bash scripts/build.sh X.Y.Z
```

### 2. 测试验证
```bash
# 1. 在测试环境部署
# 2. 运行Smoke测试
python tests/e2e/test_smoke.py

# 3. 运行集成测试
python -m pytest tests/integration/ -v

# 4. 性能验证
python tests/performance/test_batch_operations.py

# 5. 手动验证关键功能
```

### 3. 创建发布标签
```bash
# 1. 合并到主分支
git checkout main
git merge release/vX.Y.Z

# 2. 创建标签
git tag -a vX.Y.Z -m "Release X.Y.Z: 描述"

# 3. 推送标签
git push origin main --tags
```

### 4. 生成发布包
```bash
# 1. 构建发布包
bash scripts/build.sh X.Y.Z

# 2. 验证发布包内容
ls -la build-artifacts/

# 3. 测试发布包
# 在新环境中解压并验证
```

### 5. 部署发布
```bash
# 1. 备份当前环境
bash scripts/backup.sh

# 2. 部署新版本
bash scripts/deploy.sh build-artifacts/hermesnexus-X.Y.Z.tar.gz

# 3. 验证部署
bash scripts/health-check.sh

# 4. 监控日志
tail -f logs/hermesnexus.log
```

## 🔄 回滚流程

### 触发条件
- [ ] Smoke测试失败
- [ ] 核心功能异常
- [ ] 性能严重下降
- [ ] 数据一致性问题
- [ ] 安全漏洞发现

### 回滚步骤
```bash
# 1. 立即停止服务
sudo systemctl stop hermesnexus

# 2. 恢复备份
bash scripts/restore-backup.sh backup-YYYYMMDD-HHMMSS.tar.gz

# 3. 验证恢复
bash scripts/health-check.sh

# 4. 重启服务
sudo systemctl start hermesnexus

# 5. 验证功能
python tests/e2e/test_smoke.py

# 6. 通知团队
# 记录回滚原因和影响
```
