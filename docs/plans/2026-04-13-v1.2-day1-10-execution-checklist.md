# HermesNexus v1.2 Day 1-10 执行清单

Date: 2026-04-13  
Version: 1.0.0  
Status: 规划中

## 阶段目标

把 HermesNexus 从“单节点可用”推进到“多节点可管理、批量操作可用、验证流程可重复”。

## 不做范围

- 不做多协议设备接入扩展（SNMP / MQTT / NETCONF / RESTCONF）
- 不做高可用集群
- 不做跨机房 / 多云部署
- 不扩展复杂审批流
- 不做企业级大规模调度平台

## Day 1-10 清单

### Day 1：冻结范围与基线

目标：确认 v1.2 只做多节点管理 + 批量操作优化。

任务：
- [ ] 锁定 v1.2 范围和不做项
- [ ] 梳理 node / asset / task / audit / batch 术语
- [ ] 记录当前代码已有能力与缺口
- [ ] 记录本机与开发服务器当前基线

验证：
- 本机：关键单测可跑
- 开发服务器：当前服务可启动、可访问

### Day 2：冻结契约

目标：先统一对象和接口，再改代码。

任务：
- [ ] 固化节点列表 / 节点详情 / 节点状态接口
- [ ] 固化资产批量操作接口草案
- [ ] 固化任务批量分发 / 批量创建接口草案
- [ ] 统一错误码与响应结构

验证：
- 本机：schema 可导入、模型可校验
- 开发服务器：接口回放无结构冲突

### Day 3：多节点管理最小能力

目标：控制台能明确看到多个节点。

任务：
- [ ] 节点列表支持分页 / 筛选 / 状态查看
- [ ] 节点详情展示心跳、最近任务、审计摘要
- [ ] 节点在线 / 异常状态区分清晰
- [ ] 节点查询支持批量读取

验证：
- 本机：节点相关测试通过
- 开发服务器：至少 2 个节点状态可见

### Day 4：批量操作 API 骨架

目标：把单条操作提升到批量操作。

任务：
- [ ] 资产批量创建 / 批量更新 / 批量停用

- [ ] 批量结果结构统一
- [ ] 定义幂等与部分失败规则

验证：
- 本机：批量单测覆盖成功 / 失败 / 部分成功

### Day 5：Week 1 收口
