## 3. 核心概念设计

### 3.1 审计字段统一规范

```
审计字段标准:
├── 基础字段 (所有审计记录必填)
│   ├── audit_id: 审计记录ID
│   ├── event_type: 事件类型
│   ├── timestamp: 事件时间戳
│   ├── actor_id: 操作者ID
│   ├── actor_type: 操作者类型 (user/node/system)
│   └── tenant_id: 租户ID
├── 操作字段 (操作相关)
│   ├── resource_type: 资源类型
│   ├── resource_id: 资源ID
│   ├── action: 操作动作
│   ├── result: 操作结果 (success/failure)
│   └── error_message: 错误信息
└── 上下文字段 (可选)
    ├── source_ip: 来源IP
    ├── user_agent: 用户代理
    ├── correlation_id: 关联ID
    ├── metadata: 扩展元数据
    └── risk_level: 风险等级
```

### 3.2 安全链路验收标准

```
安全链路验收维度:
1. 认证链路
   ├── JWT Token生成和验证
   ├── 节点身份注册和绑定
   ├── Token过期和刷新机制
   └── 认证失败处理

2. 授权链路
   ├── 权限检查逻辑正确性
   ├── 角色权限映射完整性
   ├── 风险评估准确性
   └── 跨租户隔离有效性

3. 审批链路
   ├── 审批状态机正确转换
   ├── 审批决策记录完整性
   ├── 审批拒绝触发回滚
   └── 审批超时处理

4. 回滚链路
   ├── 回滚计划创建和执行
   ├── 故障自动分类和处理
   ├── 恢复机制有效性
   └── 回滚验证准确性

5. 审计链路
   ├── 审计日志完整性
   ├── 敏感操作记录
   ├── 审计日志查询
   └── 合规报告生成
```

### 3.3 验收测试策略

```
测试层次:
├── 单元测试 (已有)
│   └── 代码级别功能验证
├── 集成测试 (Day 5重点)
│   ├── 跨组件交互验证
│   ├── 完整流程验证
│   └── 错误场景覆盖
└── 端到端测试 (Day 5重点)
    ├── 用户场景验证
    ├── 安全边界验证
    └── 性能和可靠性验证
```
