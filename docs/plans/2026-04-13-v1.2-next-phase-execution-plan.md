# HermesNexus 下一阶段落地计划：v1.2 多节点管理与批量操作优化

Date: 2026-04-13  
Version: 1.0.0  
Status: 规划中

## 规划依据

- `README.md` 中“计划中”明确写了：v1.2（短期）= 多节点管理、批量操作优化
- `docs/41-实施里程碑与交付拆分.md` 中，下一大阶段是 Phase 4：多协议设备接入
- `docs/09-MVP开发路线图.md` 强调：本机负责开发，开发服务器负责集成验证
- `docs/68-技术栈决策与MVP边界.md` 说明：MVP 先做最小闭环，后续优先补控制面能力和可验证性

## 结论

当前 HermesNexus 已完成 MVP v1.0 和 Phase 3 安全增强，下一阶段不再重复做“能跑通”的验证，而是进入 v1.2 的控制面增强：

1. 多节点管理
2. 批量操作优化
3. 控制台批量视图与操作入口
4. 本机 / 开发服务器 / 边缘节点三层验证闭环

## 本阶段目标

把系统从“单节点可用”推进到“可管理多个节点、可批量操作、可重复验证”。

## 明确不做的事

- 不做多协议设备接入扩展（SNMP / MQTT / NETCONF / RESTCONF）
- 不做高可用集群
- 不做跨机房/多云部署
- 不扩展复杂审批流
- 不做企业级大规模调度平台
