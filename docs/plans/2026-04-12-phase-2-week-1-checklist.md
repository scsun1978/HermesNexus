# HermesNexus Phase 2 第一周开发清单

Date: 2026-04-12
Version: 1.0.0

## 目标

用一周时间把 Phase 2 的骨架先搭起来：统一对象模型、参数化部署、资产管理最小能力、任务编排最小能力、审计记录最小能力、基础控制台骨架。

## 本周原则

1. 先统一契约，再写页面。
2. 先做最小可用接口，再做美化。
3. 先保证本机可跑，再上开发服务器联调。
4. 先把烟囱拆掉，别先追求功能面铺满。
5. 每天都要有可验证产出。

## Day 1：冻结对象模型和契约

### 目标
统一 device / node / task / audit 的边界，避免后续实现反复返工。

### 任务
- 读一遍 Phase 2 相关文档：
  - `docs/41-实施里程碑与交付拆分.md`
  - `docs/09-MVP开发路线图.md`
  - `docs/68-技术栈决策与MVP边界.md`
  - `docs/v1.0/RELEASE-NOTES.md`
- 列出当前文档中所有混用的概念：jobs, tasks, devices, nodes, assets
- 确定每个对象的职责
- 写一页契约说明

### 交付物
- 对象模型说明文档
- 状态枚举和错误码草案
- 统一命名表

### 验证
- 文档里不再出现同义混用
- 所有后续任务都能按这个命名执行

## Day 2：参数化部署与配置

### 目标
去掉固定路径依赖，让本机和开发服务器使用同一套运行方式。

### 任务
- 定义环境变量规范
- 定义配置文件模板位置
- 定义本机和开发服务器的差异配置
- 将启动脚本参数化
- 统一日志目录和数据目录

### 交付物
- 配置规范文档
- 启停脚本草案
- 环境变量清单

### 验证
- 不依赖硬编码路径
- 本机和开发服务器都能用同一种方式启动

## Day 3：资产管理最小能力

### 目标
控制平面能看见“管什么”。

### 任务
- 设计资产数据结构
- 提供资产创建接口
- 提供资产列表接口
- 提供资产详情接口
- 提供资产状态查询接口

### 交付物
- 资产模型
- 资产 API 草案
- 资产页面骨架

### 验证
- 能新增一个资产
- 能列出资产
- 能查看单个资产详情

## Day 4：任务编排最小能力

### 目标
控制平面能组织任务，而不是只发单条命令。

### 任务
- 设计任务数据结构
- 提供任务创建接口
- 提供任务分派接口
- 提供任务状态查询接口
- 提供任务结果归档接口

### 交付物
- 任务模型
- 任务 API 草案
- 任务页面骨架

### 验证
- 能创建任务
- 能把任务分配到目标节点
- 能看到状态流转

## Day 5：审计记录最小能力

### 目标
所有关键动作可追踪。

### 任务
- 设计审计事件结构
- 记录任务创建、分配、执行、回传
- 提供审计查询接口
- 提供审计页面骨架

### 交付物
- 审计模型
- 审计 API 草案
- 审计页面骨架

### 验证
- 每个关键动作都能查到记录
- 审计记录能关联任务和节点

## Day 6：控制台基础骨架

### 目标
让核心状态可以被看见。

### 任务
- 建立控制台基础布局
- 建立资产页
- 建立任务页
- 建立审计页
- 建立节点状态页

### 交付物
- 控制台页面结构
- 路由草案
- 页面占位组件

### 验证
- 页面能打开
- 页面结构和后端对象一一对应

## Day 7：烟测与文档收口

### 目标
把第一周做成能回放、能验收、能交接的状态。

### 任务
- 整理 smoke 流程
- 整理 e2e 流程
- 补部署说明
- 补验收清单
- 补故障排障 Runbook

### 交付物
- Phase 2 验收清单
- smoke 流程说明
- e2e 流程说明
- Runbook 草案

### 验证
- 文档能支持别人按步骤复现
- 失败时知道去哪查

## 第一周完成标准

满足以下条件即可认为第一周达标：
- 对象模型统一
- 配置和部署参数化
- 资产能力可用
- 任务能力可用
- 审计能力可用
- 控制台有基础页面
- smoke / e2e 流程可描述
- 文档可执行

## 本周不做

- 多协议扩展
- 高可用
- 多机房
- 复杂权限
- 大规模调度
- Hermes 深度集成

## 推荐执行顺序

1. 对象模型
2. 配置参数化
3. 资产管理
4. 任务编排
5. 审计记录
6. 控制台骨架
7. smoke / e2e
8. 文档收口
