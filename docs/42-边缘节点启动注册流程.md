# 42 边缘节点启动注册流程

## 目标

定义一个 Hermes Edge Node 从启动到被云端接纳的完整流程。

## 启动流程

1. 节点进程启动。
2. 读取本地配置。
3. 检查证书与密钥。
4. 向云端注册。
5. 下载区域策略。
6. 同步设备清单。
7. 开始心跳。
8. 启动任务消费循环。

## 注册请求应包含

- node_id
- region_id
- tenant_id
- version
- capabilities
- hostname
- ip_address
- cert fingerprint

## 云端响应应包含

- registration_state
- assigned_region
- policy_version
- task_poll_interval
- feature_flags

## 注册失败处理

- 证书无效：拒绝启动并报警
- 区域不匹配：暂停并等待修复
- 云端不可达：进入离线模式
- 配置损坏：停止并要求人工介入

## 启动状态机

- init
- loading_config
- validating_identity
- registering
- syncing
- running
- degraded
- offline

## 验证点

- 注册成功后云端能看到节点
- 节点能领取任务
- 节点断网后仍能本地运行
- 恢复网络后能补传状态
