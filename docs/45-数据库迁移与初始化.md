# 45 数据库迁移与初始化

## 目标

定义云端和边缘数据库怎么初始化、怎么迁移、怎么升级。

## 云端初始化顺序

1. 创建基础库
2. 创建租户表
3. 创建区域表
4. 创建节点表
5. 创建设备表
6. 创建任务表
7. 创建事件表
8. 创建审计表

## 边缘初始化顺序

1. 创建本地配置表
2. 创建本地任务队列表
3. 创建本地缓存表
4. 创建站点记忆表
5. 初始化默认节点状态

## 迁移原则

- migration 脚本必须可重复执行
- 每次迁移都有版本号
- 迁移前必须做备份
- 迁移后必须做验证

## 验证内容

- 表是否存在
- 索引是否建立
- 默认数据是否正确
- 旧版本数据是否可读取

## 设计建议

- 云端用成熟迁移工具
- 边缘用简单、可恢复的迁移方式
- schema 变化必须兼容旧节点一段时间
