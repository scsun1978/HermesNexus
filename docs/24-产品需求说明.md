# 24 产品需求说明

## 目标

定义 Hermes 分布式边缘设备管理系统到底要解决什么问题，给后续设计和开发一个统一目标。

## 产品愿景

让一个云端控制平面 + 多个边缘 Hermes 节点，安全地管理区域内的主机、IoT 设备和网络设备，并把现场经验持续沉淀为可复用知识。

## 核心价值

- 统一管理异构设备
- 降低运维和现场操作成本
- 让边缘节点具备局部自治能力
- 提升故障发现、修复和回滚速度
- 把运维经验沉淀成技能和记忆

## 目标用户

### 1. 运维工程师
- 管理主机、网络设备、IoT 设备
- 需要巡检、批量变更、回滚、告警响应

### 2. 站点负责人
- 关注区域内设备健康
- 需要清晰的状态视图和审计记录

### 3. 平台管理员
- 关注权限、策略、审批和安全边界

### 4. 自动化系统 / Agent
- 需要通过 API 或 Hermes 技能执行任务

## 核心场景

1. 新设备接入与注册
2. 设备健康巡检
3. 批量配置变更
4. 固件升级与回滚
5. 断网边缘自治执行
6. 告警联动处置
7. 站点经验沉淀

## 成功指标

- 设备接入成功率高
- 任务执行成功率高
- 失败任务可快速回滚
- 审计链路完整
- 运维人员操作步骤减少

## 产品边界

Hermes 不是单纯监控平台，也不是通用工单系统。它的重点是：
- 理解任务
- 编排动作
- 安全执行
- 经验复用
