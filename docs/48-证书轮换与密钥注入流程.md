# 48 证书轮换与密钥注入流程

## 目标

让边缘节点与设备凭证具备可轮换、可吊销、可恢复的生命周期。

## 证书轮换对象

- 边缘节点证书
- 云边通信证书
- 设备访问凭证
- API token

## 轮换流程

1. 云端生成新凭证
2. 边缘节点拉取新凭证
3. 新旧凭证短期并行
4. 边缘切换到新凭证
5. 云端吊销旧凭证
6. 记录审计日志

## 密钥注入方式

- 环境变量
- Secret volume
- 启动时拉取
- 短期缓存

## 轮换原则

- 不要中断正在执行的任务
- 新旧凭证应有重叠窗口
- 所有轮换都要记录 trace
- 出错时可回退到旧凭证

## 失败处理

- 新凭证不可用：保留旧凭证
- 节点离线：等待恢复后轮换
- 吊销失败：上报人工介入

## 建议

- 高风险凭证优先短周期轮换
- 设备级密钥和节点级密钥分开管理
