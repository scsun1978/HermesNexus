# 13 资产模型与数据库设计

## 目标

把主机、IoT 设备、网络设备统一建模，方便调度、审计和知识沉淀。

## 核心实体

### Tenant
- 租户 / 组织

### Region
- 站点 / 区域 / 机房

### Node
- Hermes Edge Node

### Device
- 被管理设备

### Policy
- 权限与动作边界

### Job
- 高层任务

### JobStep
- 任务步骤

### Event
- 执行过程事件

### AuditRecord
- 审计记录

## Device 建模建议

字段建议：
- device_id
- tenant_id
- region_id
- type：host / iot / network
- protocol：ssh / mqtt / snmp / netconf
- hostname
- ip_address
- vendor
- model
- firmware_version
- tags
- risk_level
- last_seen_at
- status

## 关系建议

- 一个 Region 有多个 Device
- 一个 Node 管多个 Device
- 一个 Job 包含多个 JobStep
- 一个 JobStep 对应一个或多个 Event

## 数据库选型

### 云端
- PostgreSQL：主数据、任务、权限、审计
- ClickHouse / Elasticsearch：事件分析、审计检索

### 边缘
- SQLite：轻量缓存、队列、站点记忆
- 可选：RocksDB / LiteFS 做更强本地持久化

## 设计原则

- 主数据与事件数据分离
- 审计数据追加写，不轻易改写
- 设备表与拓扑表分开
- 站点级缓存可离线使用
