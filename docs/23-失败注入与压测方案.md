# 23 失败注入与压测方案

## 目标

验证 Hermes 云边架构在真实故障和大规模负载下是否稳定。

## 测试维度

### 1. 网络故障
- 断网
- 高延迟
- 丢包
- DNS 故障

### 2. 设备故障
- 设备离线
- 设备返回错误
- 认证失败
- 厂商接口超时

### 3. 边缘故障
- Hermes 节点重启
- 本地磁盘满
- 队列堆积
- 记忆写入失败

### 4. 云端故障
- API 超时
- 队列不可用
- 数据库降级
- 审批服务异常

## 压测指标

- 任务吞吐
- 平均执行时延
- P95 / P99 延迟
- 重试成功率
- 回滚成功率
- 事件补发成功率

## 推荐测试场景

### 场景 1：站点断网
- 边缘继续执行本地巡检
- 恢复后补传事件

### 场景 2：大批量重启
- 分批执行
- 验证限流与队列

### 场景 3：高危配置变更
- 检查审批链
- 检查回滚是否可用

### 场景 4：云端服务降级
- 边缘本地自治
- 减少对云端依赖

## 结论标准

- 不丢任务
- 不重复执行
- 可恢复
- 可审计
- 可定位故障

## 建议

- 先做小规模故障注入
- 再做长时间 soak test
- 最后做跨区域压测
