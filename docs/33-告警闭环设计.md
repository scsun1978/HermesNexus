# 33 告警闭环设计

## 目标

让每个告警从触发到关闭都有完整闭环，而不是只停留在通知层。

## 闭环阶段

### 1. 触发
- 监控系统触发告警
- 告警进入 Hermes

### 2. 聚合
- 汇总设备上下文
- 找到相似历史告警
- 识别站点影响范围

### 3. 分析
- 生成原因假设
- 计算风险等级
- 给出处置建议

### 4. 处置
- 生成任务
- 创建工单
- 自动化动作执行

### 5. 验证
- 检查告警是否解除
- 检查副作用
- 检查是否需要回滚

### 6. 关闭
- 记录结论
- 回写知识库
- 形成经验条目

## 闭环状态

- new
- acknowledged
- investigating
- mitigating
- resolved
- closed

## 设计建议

- 每条告警都要可追踪到设备和任务
- 每次处置都要记录依据
- 自动关闭必须有条件
- 关闭后要沉淀经验
