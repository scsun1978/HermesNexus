# 67 第一轮 Claude Code 任务拆分

## 目标

把 Hermes MVP 的第一轮开发拆成 5-8 个可执行小步，便于 Claude Code 按块落地。

这份文档只负责“第一轮任务块”，不重复定义总路线图和执行提示词。

## 任务拆分原则

- 每个任务只做一件事
- 先共享契约，后业务实现
- 先本地可跑，后云边联调
- 每个任务都要有明确验证结果
- 开发部署测试服务器固定使用 `scsun@172.16.100.101:22`

## 任务块 1：确认范围与初始化仓库

### 目标
锁定 Phase 1 MVP 范围，并把仓库骨架搭出来。

### Claude Code 要做
- 阅读 `09-MVP开发路线图.md`
- 阅读 `41-实施里程碑与交付拆分.md`
- 阅读 `58-代码仓库落地版总览.md`
- 阅读 `65-Claude Code MVP落地开发与集成步骤.md`
- 创建 monorepo 目录骨架
- 补最小 README、.env.example、docker-compose.yaml

### 验证
- 目录结构存在
- 本地开发说明可读
- 基础依赖可启动

## 任务块 2：定义共享数据契约

### 目标
冻结云端和边缘都要遵守的消息结构。

### Claude Code 要做
- 定义 Job schema
- 定义 Event schema
- 定义 Node heartbeat schema
- 定义 Device schema
- 定义错误码规范

### 验证
- 共享结构可被云端和边缘同时引用
- 至少能通过 1 个示例请求和 1 个事件样例校验

## 任务块 3：实现边缘节点最小运行时

### 目标
让边缘节点可以启动、注册、接收任务。

### Claude Code 要做
- 实现 edge node 启动入口
- 实现节点注册流程
- 实现 heartbeat 上报
- 实现任务拉取或接收接口
- 实现本地任务状态记录
- 只保留最小闭环所需字段和逻辑

### 验证
- 边缘节点能启动
- 云端能看到节点在线
- 节点能接收一个空任务或 mock 任务

## 任务块 4：实现 SSH 执行器

### 目标
让边缘节点能对 Linux 主机执行最小 SSH 命令。

### Claude Code 要做
- 创建 SSH adapter / executor
- 实现命令执行
- 实现输出捕获
- 实现超时与失败返回
- 实现最基本审计记录

### 验证
- 能对一台测试主机执行 `uptime` 或 `hostname`
- 结果能返回为结构化响应
- 失败时能记录错误

## 任务块 5：实现云端任务与事件 API

### 目标
让云端可以创建任务并查看结果。

### Claude Code 要做
- 实现创建任务 API
- 实现任务状态查询 API
- 实现事件上报 API
- 实现节点心跳查询 API
- 实现最简审计表写入

### 验证
- 云端能创建任务
- 云端能看到任务流转
- 云端能看到边缘回传事件

## 任务块 6：实现最简控制台

### 目标
让人能看到系统状态，而不是只看接口。

### Claude Code 要做
- 做一个总览页
- 显示节点在线状态
- 显示任务列表与状态
- 显示最近事件或错误
- 只做最简可见，不做完整交互

### 验证
- 浏览器能打开控制台
- 能看到节点和任务状态
- 能从 UI 观察到任务回传结果

## 任务块 7：补模拟器与联调脚本

### 目标
在没有真实站点的情况下完成完整链路验证。

### Claude Code 要做
- 写 Linux 主机模拟器或最小测试宿主
- 写边缘节点联调脚本
- 写任务下发模拟脚本
- 写故障场景模拟脚本
- 模拟器优先服务于闭环验证，不追求完备性

### 验证
- 不依赖真实生产设备也能跑通闭环
- 可重复执行

## 任务块 8：补测试、回滚、验收

### 目标
给 MVP 的最小闭环加上可验证性。

### Claude Code 要做
- 写核心单元测试
- 写集成测试
- 写端到端测试
- 写发布前验收清单
- 写回滚说明
- 测试优先覆盖最小闭环和失败恢复

### 验证
- 测试可运行
- 回滚步骤明确
- 验收清单能覆盖最小闭环

## 推荐执行顺序

1. 任务块 1：确认范围与初始化仓库
2. 任务块 2：定义共享数据契约
3. 任务块 3：实现边缘节点最小运行时
4. 任务块 4：实现 SSH 执行器
5. 任务块 5：实现云端任务与事件 API
6. 任务块 6：实现最简控制台
7. 任务块 7：补模拟器与联调脚本
8. 任务块 8：补测试、回滚、验收

## 每个任务块的 Claude Code 输出要求

- 改了哪些文件
- 做了什么实现
- 用什么命令验证
- 验证结果是什么
- 是否还有未完成项

## MVP 第一轮的完成定义

当以下链路跑通时，第一轮完成：

云端创建任务 → 边缘接收 → SSH 执行 → 结果回传 → 控制台可见
