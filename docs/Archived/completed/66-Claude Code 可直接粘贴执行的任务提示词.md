# 66 Claude Code 可直接粘贴执行的任务提示词

你现在是 Hermes 分布式边缘设备管理系统的实现助手。请基于以下约束，开始实现 MVP。

## 背景

- 这是一个云边协同的分布式设备管理系统
- 目标是先做最小可验证 MVP
- 只做 Phase 1：单边缘节点验证
- 设备类型先只支持 Linux 主机
- 协议先只支持 SSH
- 最小闭环是：云端创建任务 → 边缘接收 → SSH 执行 → 结果回传 → 云端展示

## 本轮执行边界

- 只做 Phase 1 的最小闭环。
- 只支持 Linux 主机和 SSH。
- 先实现云端创建任务、边缘接收任务、SSH 执行、结果回传、云端可见。
- 不要扩展到多协议、多设备类型、高可用、复杂权限。
- 如果仓库骨架未就绪，先补骨架，不要提前深入业务实现。
- 本次 MVP 的开发部署测试服务器为：`scsun@172.16.100.101:22`。

## 开发原则

- 先搭骨架，再做闭环
- 先共享契约，再写实现
- 先最小可运行，再加功能
- 先本地联调，再考虑生产化
- 任何写操作都要可审计

## 你的工作顺序

### Step 1：读取并总结 MVP 范围
- 识别本次只做 Phase 1
- 列出本次 MVP 的边界和不做的内容
- 输出一个简短总结

### Step 2：搭建仓库骨架
- 按 monorepo 思路组织目录
- 准备 cloud / edge / shared / drivers / console / deploy / tests / docs
- 准备 `.env.example`、`docker-compose.yaml`、`README.md` 的最小骨架

### Step 3：定义共享契约
- 定义 Job schema
- 定义 Event schema
- 定义 Node heartbeat schema
- 定义 Device schema
- 定义 machine-readable error code

### Step 4：实现边缘最小闭环
- 边缘节点可注册
- 能拉取或接收任务
- 能执行 SSH 命令
- 能回传结果
- 能记录本地审计

### Step 5：实现云端最小控制面
- 节点注册 API
- 任务创建 API
- 任务状态查询 API
- 事件查询 API
- 最简控制台展示

### Step 6：加测试和模拟器
- 节点模拟器
- SSH 设备模拟器
- 联调脚本
- 基础单元测试 / 集成测试

### Step 7：收口交付
- 输出当前完成内容
- 输出待办事项
- 输出下一步建议

## 你每次输出都要包含

- 做了什么
- 改了哪些文件
- 怎么验证
- 验证结果
- 下一步

## 开始执行时的第一个动作

先扫描仓库结构和现有文档，确认代码仓库现状，然后给出一个可执行的 MVP 实施起点。

如果发现 `09-MVP开发路线图.md` 和 `67-第一轮Claude Code任务拆分.md` 的边界与当前仓库实现不一致，优先按这两个文档收口任务范围，再开始写代码。

## 成功标准

只要能完成下面这一条链路，就算 MVP 第一阶段成功：

云端创建任务 → 边缘节点接收 → SSH 执行 → 结果回传 → 云端可见
