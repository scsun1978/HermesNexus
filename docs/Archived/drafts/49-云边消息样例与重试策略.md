# 49 云边消息样例与重试策略

## 目标

给云边消息通道提供可直接实现的样例和重试规则。

## 消息样例：任务派发

```json
{
  "message_id": "msg-001",
  "trace_id": "trace-001",
  "type": "task.assign",
  "sender_id": "cloud",
  "receiver_id": "edge-01",
  "payload": {
    "job_id": "job-123",
    "region_id": "r1",
    "steps": []
  }
}
```

## 消息样例：执行结果

```json
{
  "message_id": "msg-002",
  "trace_id": "trace-001",
  "type": "task.finished",
  "sender_id": "edge-01",
  "receiver_id": "cloud",
  "payload": {
    "job_id": "job-123",
    "status": "succeeded"
  }
}
```

## 重试策略

- 可重试：网络超时、临时不可达、队列拥塞
- 不可重试：签名失败、权限拒绝、参数非法
- 重试间隔采用指数退避
- 重试次数设上限

## 去重策略

- message_id 唯一
- receiver 侧保存已处理 ID
- 重复消息直接 ACK，不重复执行

## 设计建议

- 所有消息都应可落地到审计表
- 边缘节点要有本地 outbox
- 云端要有 dead-letter 处理机制
