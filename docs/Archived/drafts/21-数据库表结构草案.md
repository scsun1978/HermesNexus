# 21 数据库表结构草案

## 目标

给云端与边缘存储设计一个最小但完整的表结构草案。

## 云端表

### tenants
- id
- name
- created_at

### regions
- id
- tenant_id
- name
- description
- created_at

### nodes
- id
- region_id
- name
- status
- last_seen_at
- version

### devices
- id
- region_id
- node_id
- type
- protocol
- hostname
- ip_address
- vendor
- model
- firmware_version
- risk_level
- status
- last_seen_at

### policies
- id
- tenant_id
- name
- rules_json
- enabled

### jobs
- id
- tenant_id
- region_id
- submitted_by
- status
- priority
- rollback_enabled
- created_at
- updated_at

### job_steps
- id
- job_id
- step_index
- action
- target_device_id
- params_json
- status
- started_at
- finished_at

### events
- id
- job_id
- node_id
- device_id
- type
- payload_json
- created_at

### audit_records
- id
- actor
- action
- target
- decision
- reason
- created_at

## 边缘表

### local_tasks
- id
- job_id
- payload_json
- status
- retry_count
- next_retry_at

### local_events
- id
- event_type
- payload_json
- created_at

### local_cache_devices
- device_id
- snapshot_json
- updated_at

### local_memory_entries
- id
- category
- content
- updated_at

## 设计原则

- 事件表追加写
- 任务表状态机化
- 缓存表可丢失、可重建
- 审计表不可随意修改
