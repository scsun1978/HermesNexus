# 44 接口契约样例

## 目标

给云端与边缘之间的接口一个可执行的契约范式。

## 约定

- 请求必须带 request_id
- 写操作必须幂等
- 响应必须返回 success、error、trace_id
- 所有接口都要可记录审计

## 示例：创建任务

```http
POST /jobs
Content-Type: application/json
```

```json
{
  "request_id": "req-123",
  "tenant_id": "t1",
  "region_id": "r1",
  "name": "nightly-health-check",
  "target": "site-a",
  "priority": "medium",
  "rollback_enabled": true,
  "steps": [
    {
      "name": "precheck",
      "action": "read_status"
    },
    {
      "name": "collect",
      "action": "fetch_logs"
    }
  ]
}
```

## 示例：节点心跳

```http
POST /nodes/edge-01/heartbeat
```

```json
{
  "request_id": "hb-001",
  "node_id": "edge-01",
  "status": "running",
  "queue_depth": 3,
  "load": {
    "cpu": 0.42,
    "memory": 0.61
  },
  "capabilities": ["ssh-host", "mqtt-device", "snmp-network"]
}
```

## 示例：事件上报

```json
{
  "request_id": "evt-001",
  "trace_id": "trace-abc",
  "event_type": "job.step.failed",
  "payload": {
    "job_id": "job-123",
    "step": "restart_service",
    "error": "timeout"
  }
}
```

## 契约要点

- 参数字段名必须稳定
- error 字段必须可机器解析
- 成功和失败都要返回 trace_id
- 同类接口返回结构尽量一致
