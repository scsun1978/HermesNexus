# 71 开发启动 Checklist

## 目标

把 HermesNexus 的开发前置条件整理成一份可以直接打勾执行的启动清单，方便在正式开发前快速确认是否具备开工条件。

## 使用方式

按顺序检查；全部满足后再进入 MVP 开发。

## A. 必须先完成

### 仓库与环境

- [ ] 仓库目录结构已确定
- [ ] cloud / edge / shared / console / deploy / tests / docs 主目录已就位
- [ ] Python 虚拟环境已创建
- [ ] 前端项目初始化方案已确定
- [ ] Makefile / scripts 入口已确定

### 本机隔离

- [ ] HermesNexus 使用独立虚拟环境
- [ ] HermesNexus 与当前主 Hermes 环境隔离
- [ ] 已明确项目级 `HERMES_HOME`
- [ ] 已有项目专用 `.env`
- [ ] 已有项目专用 `config.yaml` 模板

### 开发部署测试服务器

- [ ] 开发部署测试服务器已明确：`scsun@172.16.100.101:22`
- [ ] SSH 可达
- [ ] 部署账号权限足够
- [ ] Docker / Docker Compose 可用
- [ ] 必要端口已规划
- [ ] 防火墙 / 安全组策略已确认

### 技术栈

- [ ] 云端后端技术栈已定
- [ ] 边缘运行时技术栈已定
- [ ] 共享协议层技术栈已定
- [ ] 控制台前端技术栈已定
- [ ] 部署方式已定为 Docker Compose 起步

### 配置与密钥

- [ ] `.env.example` 已有
- [ ] `config.example.yaml` 已有
- [ ] secrets 不入 git
- [ ] 证书、Token、SSH key 的管理方式已明确
- [ ] 云端 / 边缘 / 本机的配置边界已明确

## B. 可以并行准备

### 最小数据模型

- [ ] Node 模型
- [ ] Job 模型
- [ ] Event 模型
- [ ] Device 模型
- [ ] Audit 模型
- [ ] Memory provider 接口

### 运行和验证方式

- [ ] 一键启动命令已定义
- [ ] 一键停止命令已定义
- [ ] 一键测试命令已定义
- [ ] 一键清理命令已定义
- [ ] 最小 smoke test 已定义

### 质量基线

- [ ] 格式化工具已确定
- [ ] lint 工具已确定
- [ ] 单元测试框架已确定
- [ ] 集成测试框架已确定
- [ ] 最低验收标准已确定

### 观测与排障

- [ ] 结构化日志格式已确定
- [ ] trace_id / request_id 策略已确定
- [ ] health endpoint 已确定
- [ ] 基础故障排查流程已确定

## C. 可以延后

- [ ] OpenViking 深度集成
- [ ] 多协议设备全覆盖
- [ ] 多区域调度
- [ ] 高可用集群
- [ ] 复杂审批流
- [ ] 全量知识图谱
- [ ] Kubernetes 生产编排

## D. 开发启动门槛

只有下面这些项全部满足，才建议正式进入 MVP 开发：

- [ ] 仓库骨架明确
- [ ] 本机开发隔离明确
- [ ] 开发部署测试服务器可用
- [ ] 技术栈明确
- [ ] 配置和密钥边界明确
- [ ] 最小数据模型明确
- [ ] 启动和测试命令明确
- [ ] 质量基线明确
- [ ] 观测和排障路径明确

## E. 开工顺序建议

1. 先确认仓库骨架
2. 再确认本机隔离
3. 再确认开发服务器
4. 再确认技术栈
5. 再确认配置和密钥
6. 再确认数据模型
7. 再确认命令和测试
8. 最后开始写业务代码
