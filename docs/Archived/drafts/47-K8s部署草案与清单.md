# 47 K8s 部署草案与清单

## 目标

把 Hermes 云端服务与边缘节点都变成可管理、可滚动升级的 Kubernetes 工作负载。

## 云端工作负载

- Deployment: cloud-api
- Deployment: task-orchestrator
- Deployment: asset-service
- Deployment: event-service
- Deployment: console
- StatefulSet: postgres
- StatefulSet: redis / nats

## 边缘工作负载

- Deployment: edge-node
- 可选：DaemonSet 部署到每个边缘节点

## K8s 资源清单

### 基础资源
- Namespace
- ConfigMap
- Secret
- Service
- Ingress
- PersistentVolumeClaim

### 运维资源
- HPA
- PodDisruptionBudget
- NetworkPolicy
- CronJob（用于巡检和备份）

## 部署建议

- 云端和边缘分 namespace 管理
- Secret 使用外部密钥系统注入
- 重要服务至少保留两个副本
- 边缘节点可按区域分组部署

## 验证项目

- 滚动升级是否成功
- 证书是否正确注入
- 节点是否能自动重连云端
- 故障恢复是否正常
