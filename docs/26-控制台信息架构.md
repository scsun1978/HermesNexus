# 26 控制台信息架构

## 目标

定义云端 Web 控制台应该如何组织信息，让用户能快速定位资产、任务、风险和结果。

## 主导航建议

- 总览
- 资产
- 区域
- 任务
- 告警
- 审计
- 记忆
- 策略
- 节点
- 设置

## 页面结构

### 1. 总览
- 在线节点数
- 设备健康概览
- 最近任务成功率
- 未处理告警
- 高风险变更

### 2. 资产页
- 设备列表
- 标签筛选
- 区域筛选
- 协议筛选
- 风险等级筛选

### 3. 任务页
- 任务状态列表
- 任务详情
- 步骤执行过程
- 回滚信息

### 4. 告警页
- 当前告警
- 告警历史
- 告警联动任务

### 5. 审计页
- 谁发起了什么
- 谁批准了什么
- Hermes 做了什么
- 结果如何

### 6. 记忆页
- 站点知识
- 故障经验
- 技能推荐
- 最近学习到的规则

## 可视化原则

- 先总览，再深入
- 风险信息突出显示
- 失败信息和回滚入口明显
- 所有列表都支持过滤和搜索

## Hermes 对控制台的作用

Hermes 不只是被展示，它还应能：
- 理解控制台上的任务输入
- 根据页面上下文建议动作
- 自动补全任务草案
- 在失败时生成说明和下一步建议
