# 57 开发节奏与日常工作方式

## 目标

让开发、联调、上线、复盘形成稳定节奏。

## 日常节奏

### 每日
- 看任务进展
- 看告警与失败项
- 处理阻塞
- 更新文档

### 每周
- 做一次版本同步
- 做一次回归
- 做一次故障复盘
- 更新里程碑

### 每阶段结束
- 验证交付物
- 检查验收标准
- 整理经验进入知识库

## 工作方式建议

- 文档先行
- 接口先定
- 测试先写
- 发布有回滚
- 故障有 Runbook

## 团队协作原则

- 一个问题只保留一个权威结论
- 所有决策都要有记录
- 不要靠口头记忆推进关键系统
- 经验要沉淀到文档和 skill
