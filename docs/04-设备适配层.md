# 04 设备适配层

## 目标

把不同协议、不同设备类型统一成一个动作模型。

## 三类适配器

### 1. 主机适配器
支持：
- SSH
- WinRM
- 本地 agent

常见能力：
- 执行命令
- 文件分发
- 服务重启
- 日志收集
- 配置检查

### 2. IoT 适配器
支持：
- MQTT
- HTTP / REST
- CoAP
- Modbus

常见能力：
- 状态读取
- 遥测订阅
- OTA 升级
- 控制指令
- 规则联动

### 3. 网络设备适配器
支持：
- SSH
- NETCONF
- RESTCONF
- SNMP

常见能力：
- 配置读取
- 配置备份
- 差异比对
- 回滚
- 接口状态检查

## 统一动作模型

每个动作都应包含：
- target
- verb
- parameters
- precheck
- execute
- verify
- rollback
- audit

## 设计原则

- 适配器只做协议翻译，不做业务决策
- 业务决策交给 Hermes / Skill
- 每个适配器都要支持幂等和超时
- 高危操作必须具备验证与回滚
