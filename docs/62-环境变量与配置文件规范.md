# 62 环境变量与配置文件规范

## 目标

把 Hermes 的配置、密钥和环境信息分层管理，避免散落和混乱。

## 配置层次

### 1. 环境变量
适合：
- secrets
- 运行时开关
- 连接地址

### 2. 配置文件
适合：
- 服务参数
- 功能开关
- 默认值

### 3. 本地持久化文件
适合：
- 站点记忆
- 缓存
- 队列
- 证书

## 推荐文件

- .env
- config.yaml
- openapi.yaml
- docker-compose.yaml
- k8s/*.yaml

## 命名原则

- 大写下划线用于环境变量
- YAML 适合层级结构
- secrets 不进 git
- 配置文件可模板化

## 配置分类

### 云端
- DATABASE_URL
- REDIS_URL
- JWT_SECRET
- API_BASE_URL

### 边缘
- HERMES_EDGE_ID
- HERMES_REGION_ID
- CLOUD_ENDPOINT
- NODE_CERT_PATH

### 设备驱动
- SSH_USER
- SSH_KEY_PATH
- MQTT_BROKER_URL
- SNMP_COMMUNITY

## 设计建议

- 提供 .env.example
- 提供 config.example.yaml
- secrets 单独管理
- 配置变更要支持热更新或滚动重启
