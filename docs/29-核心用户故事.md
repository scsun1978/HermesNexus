# 29 核心用户故事

## 目标

把 Hermes 产品的关键使用路径具体化，方便后续做 PRD、原型和验收标准。

## 用户故事 1：新增设备接入

作为平台管理员，我希望把新设备快速接入到 Hermes 中，这样我可以统一纳管站点资产。

验收点：
- 支持手动添加和自动发现
- 支持按区域绑定
- 支持协议探测
- 接入结果可审计

## 用户故事 2：日常巡检

作为运维工程师，我希望一键发起巡检任务，这样我可以快速知道站点健康情况。

验收点：
- 自动生成巡检步骤
- 汇总设备健康结果
- 支持异常高亮
- 支持导出报告

## 用户故事 3：批量变更

作为运维工程师，我希望对一批设备执行配置变更，这样我可以减少重复劳动。

验收点：
- 支持分批执行
- 支持审批
- 支持预检
- 支持回滚

## 用户故事 4：告警联动

作为站点负责人，我希望告警可以自动触发分析和建议，这样我可以更快定位问题。

验收点：
- 告警自动汇总上下文
- Hermes 生成建议
- 可创建工单
- 可自动发起部分修复动作

## 用户故事 5：离线自治

作为现场运维人员，我希望边缘节点断网时仍可执行本地任务，这样我不会因为云端不可用而停工。

验收点：
- 本地任务缓存
- 离线继续执行
- 恢复后补传结果
- 状态一致性可恢复

## 用户故事 6：经验沉淀

作为系统管理员，我希望系统把处理过的故障总结成知识和技能，这样后续同类问题可以复用。

验收点：
- 自动生成摘要
- 可转成 skill
- 可写入记忆
- 可搜索复用
