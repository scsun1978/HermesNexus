# 16 边缘节点部署与运维

## 目标

把 Hermes Edge Node 作为可复制、可升级、可监控、可回滚的标准化单元部署到各个区域。

## 部署形态

### 1. 单机部署
适合小站点、测试环境。

### 2. 容器部署
适合标准化交付。

### 3. VM 部署
适合传统企业环境。

### 4. K8s / Nomad 部署
适合大规模区域集群。

## 节点启动流程

1. 节点生成身份
2. 向云端注册
3. 拉取策略与配置
4. 同步设备清单
5. 启动本地队列和适配器
6. 开始心跳

## 运维任务

- 节点升级
- 配置下发
- 证书轮换
- 日志轮转
- 故障恢复

## 节点健康检查

- 程序进程状态
- 连接云端状态
- 本地队列积压
- 适配器健康状态
- 最近错误

## 升级建议

- 蓝绿升级
- 分批 rollout
- 失败自动回滚
- 保留旧版本可切换

## 运维原则

- 边缘节点尽量无状态化，状态放本地持久层
- 关键状态要可备份
- 节点升级不能影响本地任务执行
