# 59 仓库目录与模块边界

## 推荐仓库结构

```text
hermes-platform/
├── cloud/
├── edge/
├── drivers/
├── shared/
├── console/
├── deploy/
├── scripts/
├── tests/
└── docs/
```

## 目录职责

### cloud/
云端控制平面：
- identity-service
- asset-service
- task-service
- policy-service
- event-service
- memory-service
- api-gateway

### edge/
边缘节点运行时：
- Hermes runtime
- 本地队列
- 设备调度器
- 同步代理

### drivers/
设备驱动与协议适配：
- ssh
- mqtt
- snmp
- netconf
- restconf

### shared/
共享协议与模型：
- Job schema
- Event schema
- message schema
- error codes
- SDK types

### console/
控制台前端：
- 总览页
- 资产页
- 任务页
- 告警页
- 审计页

### deploy/
部署和运维：
- docker-compose
- k8s manifests
- env templates
- release scripts

### scripts/
辅助脚本：
- init
- migrate
- seed
- simulate
- backup

### tests/
- 单元测试
- 集成测试
- 端到端测试
- 模拟器测试

### docs/
- 架构文档
- API 文档
- runbook
- 设计说明

## 模块边界原则

1. shared 只放稳定协议，不放业务实现。
2. drivers 只处理协议细节，不处理权限决策。
3. edge 只管本地执行，不直接承担全局治理。
4. cloud 只管编排与治理，不直接耦合设备协议。
5. console 只面向用户交互，不承载核心逻辑。

## 目录演进建议

- MVP：单仓库 + 共享协议目录
- 规模化：cloud/edge/drivers 可拆独立仓库
- 稳定后：shared 作为单独 SDK 仓库
