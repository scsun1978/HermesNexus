# 07 安全与审计

## 安全目标

- 最小权限
- 可追溯
- 可撤销
- 可隔离
- 可审批

## 核心机制

### 1. 身份认证
- 每个边缘节点唯一身份
- 设备证书与短期 token
- mTLS 连接云边通道

### 2. 授权控制
- 区域隔离
- 设备类型隔离
- 动作级白名单
- 高危动作审批

### 3. 命令安全
- 禁止任意 shell 直通云端执行
- 所有动作必须经过 Task / Skill 层
- 支持执行前预检和执行后验证

### 4. 审计链路
- 谁发起
- 谁批准
- Hermes 如何拆解
- 实际执行了什么
- 执行结果如何
- 是否回滚

### 5. 密钥管理
- 云端集中管理
- 边缘只使用短期凭证
- 轮换与吊销机制

## 风险分级

- 低风险：读状态、采集指标
- 中风险：重启、重连、刷新配置
- 高风险：升级、批量改配置、回滚生产设备

## 审计建议

- 每个 JobStep 单独记录
- 每个设备单独记录最终状态
- 每次失败都要保留证据
- 支持从审计日志重建执行路径
