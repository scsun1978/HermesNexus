# Hermes 分布式边缘设备管理系统

> 目标：基于 Hermes 设计一个云边协同的分布式管理系统，用于管理区域内的主机、IoT 设备和网络设备。

## 总览

- 云端：集中式控制平面，负责资产、策略、任务编排、审计、告警。
- 边缘：部署 Hermes Edge Node，负责本地执行、离线缓存、设备适配、局部自治。
- 设备层：通过统一 Adapter 抽象主机、IoT、网络设备。
- 记忆层：Hermes 内置记忆 + 外部记忆 provider + 云端知识库。

## 目标能力

- 统一接入多类型设备
- 云端下发策略与任务
- 边缘自治执行与失败回滚
- 全链路审计与可观测
- 经验沉淀为可复用技能和记忆

## 设计原则

1. 云端管策略，不做逐设备重执行。
2. 边缘管执行，不依赖长时间在线。
3. 所有动作可审计、可回滚、可限权。
4. 所有设备能力通过 Adapter / Skill 暴露。
5. 知识沉淀分层：短期上下文、站点记忆、全局知识。
