# 35 商业价值与落地范围

## 目标

说明 Hermes 分布式边缘设备管理系统为什么值得做，以及最先落地的范围应该是什么。

## 商业价值

- 降低人工运维成本
- 提升故障响应速度
- 降低误操作风险
- 支持大规模异构设备纳管
- 把知识从人脑转为系统能力

## 直接收益

- 减少重复巡检
- 减少手工变更
- 减少告警遗漏
- 缩短恢复时间
- 提升站点复制效率

## 适合切入的场景

### 1. 多站点运维
- 每个站点都有独立设备集合
- 需要边缘自治与统一治理

### 2. 异构设备较多的场景
- 主机、IoT、网络设备混合存在
- 协议复杂，需要统一抽象

### 3. 远程或弱网现场
- 云端不稳定
- 边缘节点必须能自治

## 不建议一开始就做的事

- 全协议全厂商一口气支持
- 过度复杂的可视化大屏
- 过早做非常细的 AI 自动修复闭环
- 试图一次性替换所有现有运维系统

## 落地范围建议

第一阶段聚焦：
- Linux 主机
- 1-2 种 IoT 协议
- 基础网络设备读写
- 巡检、重启、升级、回滚
- 审批、审计、告警联动

## 产品判断标准

如果 Hermes 能够：
- 让一个站点运维更省人
- 让一次故障恢复更快
- 让经验更容易复用

那么它就是有价值的。
