# 01 总体架构

## 架构分层

### 1. 云端控制平面
负责全局治理：
- 设备注册与身份管理
- 任务编排与下发
- 策略、权限、审批
- 资产与拓扑管理
- 监控、日志、审计

### 2. 边缘 Hermes 节点
每个区域、站点或机房部署一个或多个 Hermes 节点：
- 接收任务
- 本地执行
- 离线缓存
- 失败重试
- 站点级记忆

### 3. 设备适配层
将不同协议统一成标准动作模型：
- 主机：SSH / WinRM / Agent
- IoT：MQTT / HTTP / CoAP / Modbus
- 网络设备：SSH / NETCONF / RESTCONF / SNMP

### 4. 数据与记忆层
- 状态数据：资产、任务、事件、审计
- 记忆数据：Hermes MEMORY.md / USER.md、外部 memory provider、云端知识库

## 运行路径

1. 云端生成任务。
2. 任务分配给指定区域的 Hermes 节点。
3. 边缘节点调用对应 Adapter / Skill。
4. 执行结果回传云端。
5. 关键经验沉淀为记忆和技能。

## 推荐拓扑

- 云端控制平面：1 套
- 每区域 Edge Node：1-3 个
- 每类设备：对应一个或多个 Adapter
- 高风险任务：必须经过审批流

## 关键设计点

- 边缘优先：断网时仍能执行本地任务
- 状态最终一致：云端不要求每次实时同步完成
- 能力插件化：新增设备协议不改主流程
- 人类可控：高危动作必须可拦截
