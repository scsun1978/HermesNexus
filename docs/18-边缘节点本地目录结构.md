# 18 边缘节点本地目录结构

## 目标

让每个 Hermes Edge Node 的本地状态有清晰、可备份、可迁移的目录布局。

## 建议目录

```text
~/.hermes-edge/
├── config/
├── state/
├── cache/
├── memory/
├── skills/
├── logs/
├── queue/
├── backups/
└── secrets/
```

## 目录说明

### config/
- 本地节点配置
- 区域信息
- 云端连接参数

### state/
- 节点状态
- 设备快照
- 最近任务状态

### cache/
- 临时缓存
- 设备信息缓存
- API 响应缓存

### memory/
- 站点级 MEMORY.md
- USER.md
- 本地知识摘要

### skills/
- 站点专用技能
- 设备类型专用技能
- 运维脚本包装

### logs/
- 执行日志
- 错误日志
- 审计摘要

### queue/
- 待执行任务
- 离线待发事件

### backups/
- 配置快照
- 设备备份
- 回滚包

### secrets/
- 本地敏感信息
- 证书缓存
- 短期 token

## 原则

- secrets 只存最少必要内容
- queue 必须持久化
- memory 必须可审查
- backups 必须有版本号
