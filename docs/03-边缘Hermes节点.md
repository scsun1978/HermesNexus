# 03 边缘 Hermes 节点

## 角色定位

边缘节点是“智能执行体”，不是纯转发器。

它负责：
- 理解任务
- 选择技能
- 调用适配器
- 本地执行
- 回传结果
- 失败恢复
- 经验沉淀

## 组成模块

### 1. Hermes Runtime
- 运行 Hermes Agent
- 接收任务与上下文
- 进行步骤编排与推理

### 2. Local Task Runner
- 真正执行系统命令、设备协议、脚本
- 负责超时、重试、并发控制

### 3. Local State Store
- 本地任务队列
- 最近执行结果
- 离线待发事件
- 缓存资产快照

### 4. Local Memory
- 站点级经验
- 设备级习惯
- 故障模式
- 常见处理流程

### 5. Sync Agent
- 与云端保持长连接
- 断网可继续工作
- 恢复后补报事件

## 边缘节点的特点

- 小范围高自治
- 对站点内设备有更低延迟
- 可承受云端短时不可用
- 可把“现场经验”快速沉淀下来

## 边缘节点的典型职责

- 夜间巡检
- 配置备份
- 固件升级
- 告警处置
- 定时健康检查
- 局部自动修复
