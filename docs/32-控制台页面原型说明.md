# 32 控制台页面原型说明

## 目标

定义控制台每个页面应该展示什么信息，便于后续设计 UI 原型。

## 页面 1：总览页

展示：
- 在线节点数
- 在线设备数
- 异常设备数
- 待审批任务数
- 当前告警数
- 最近 24 小时任务成功率

## 页面 2：资产页

展示：
- 设备列表
- 筛选条件
- 设备详情侧栏
- 协议类型
- 风险等级
- 最近心跳

## 页面 3：任务页

展示：
- 任务列表
- 任务状态
- 步骤轨迹
- 回滚入口
- 执行日志

## 页面 4：告警页

展示：
- 当前告警
- 历史告警
- 告警等级
- 关联设备
- 建议动作

## 页面 5：记忆页

展示：
- 站点经验摘要
- 最近学习到的规则
- 可复用技能
- 搜索框

## 页面 6：策略与权限页

展示：
- 风险分级规则
- 审批流配置
- 角色矩阵
- 白名单

## 原型设计原则

- 左侧导航，右侧详情
- 列表支持搜索和筛选
- 风险与失败状态颜色突出
- 每个详情页都有“下一步动作”入口

## Hermes 在页面中的作用

Hermes 可在页面中提供：
- 自然语言下发任务
- 自动解释状态
- 自动生成摘要
- 自动推荐下一步动作
