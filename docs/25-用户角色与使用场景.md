# 25 用户角色与使用场景

## 目标

把不同用户在系统里的目标、权限和常见操作讲清楚。

## 用户角色

### 1. 运维工程师
职责：
- 巡检设备
- 处理告警
- 变更配置
- 升级和回滚

关心：
- 快速执行
- 风险可控
- 结果可验证

### 2. 平台管理员
职责：
- 管理租户、区域、策略、审批
- 维护设备接入规范
- 审计系统行为

关心：
- 安全边界
- 权限模型
- 可追踪性

### 3. 站点负责人
职责：
- 查看区域健康
- 跟踪重大故障
- 评估现场风险

关心：
- 一眼看到状态
- 重大风险提醒
- 站点级健康趋势

### 4. 自动化 Agent
职责：
- 接收任务
- 调用 API
- 触发编排
- 回传结果

关心：
- 稳定接口
- 清晰语义
- 低歧义的任务模型

## 典型场景

### 场景 1：夜间巡检
- 运维工程师发起巡检任务
- Hermes 自动拆解为多个设备检查步骤
- 结果汇总成健康报告

### 场景 2：批量升级
- 管理员审批升级计划
- 边缘节点分批执行
- 每一批都要验证和可回滚

### 场景 3：断网现场处理
- 边缘节点在离线状态下继续执行本地修复
- 恢复后统一回传事件

### 场景 4：异常告警联动
- 监控系统触发事件
- Hermes 生成分析和处置建议
- 必要时执行自动化动作

## 设计启示

- 用户不是只要“一个按钮”，而是要“可控、可解释、可恢复”的自动化能力
- 对运维人员来说，系统必须减少重复劳动
- 对管理员来说，系统必须可治理
