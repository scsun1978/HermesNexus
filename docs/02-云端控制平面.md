# 02 云端控制平面

## 角色

云端是系统的大脑与治理中心，不直接承担全部设备执行。

## 核心服务

### 1. Identity Service
- 节点注册
- 证书签发
- 区域 / 租户 / 角色管理
- mTLS 与 token 轮换

### 2. Asset Service
- 设备主数据
- 标签、区域、位置
- 主机 / IoT / 网络设备分类
- 拓扑关系

### 3. Policy Service
- 权限边界
- 高危动作审批
- 命令白名单
- 回滚策略

### 4. Task Orchestrator
- 接收高层目标
- 拆解为子任务
- 分配给边缘节点
- 管理重试、超时、补偿

### 5. Telemetry Service
- 心跳
- 事件流
- 指标
- 日志
- 告警

### 6. UI / API Gateway
- Web 控制台
- OpenAPI / gRPC
- 与工单系统、告警系统联动

## 数据模型建议

- Tenant
- Region
- Node
- Device
- Policy
- Job
- JobStep
- Event
- AuditRecord

## 云端职责边界

云端要做：
- 全局视图
- 策略分发
- 风险治理
- 任务编排
- 审计存档

云端不要做：
- 每条命令逐设备直连执行
- 临时状态只保留在内存
- 让业务逻辑绑定单个协议实现
