# 37 服务拆分与接口定义

## 目标

把系统拆成边界清晰的服务，并定义它们之间的接口契约。

## 服务拆分

### 1. Identity Service
- 节点身份
- 设备身份
- 证书与 token

### 2. Asset Service
- 资产登记
- 标签管理
- 拓扑关系

### 3. Policy Service
- 风险策略
- 审批规则
- 白名单

### 4. Task Service
- 创建 Job
- 拆解 Step
- 状态机管理

### 5. Edge Gateway Service
- 云边通信
- 节点心跳
- 任务派发

### 6. Event Service
- 事件收集
- 审计存档
- 检索查询

### 7. Memory Service
- 站点记忆
- 故障经验
- 技能推荐

### 8. Console Service
- Web 界面
- 管理 API
- 用户交互

## 接口分层

### 1. 外部 API
面向控制台、自动化系统、第三方平台。

### 2. 内部 RPC / gRPC
面向服务间调用。

### 3. 边缘消息接口
面向 Edge Node。

## 接口设计原则

- 请求和响应必须结构化
- 所有写操作要有幂等键
- 所有任务要有 trace_id
- 服务之间尽量少耦合

## 示例接口

### 创建任务
- POST /jobs

### 查询设备
- GET /devices/{id}

### 节点心跳
- POST /nodes/{id}/heartbeat

### 事件上报
- POST /events

## 建议

- 先从 5 个核心服务开始：身份、资产、任务、边缘网关、事件
- 记忆和控制台可在第二阶段增强
