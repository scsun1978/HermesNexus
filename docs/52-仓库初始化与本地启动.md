# 52 仓库初始化与本地启动

## 目标

让新开发者可以在最短时间内把 Hermes 项目跑起来。

## 初始化步骤

1. 克隆仓库。
2. 安装依赖。
3. 复制环境变量模板。
4. 启动基础依赖服务。
5. 启动云端服务。
6. 启动边缘节点模拟器。
7. 打开控制台验证。

## 推荐启动顺序

### 本地开发环境
- postgres
- redis 或 nats
- cloud-api
- task-orchestrator
- edge-node
- console

### 验证顺序

1. 数据库是否可连接
2. API 是否可访问
3. 节点是否注册成功
4. 任务是否能派发
5. 事件是否能回传

## 本地开发原则

- 默认使用 Docker Compose
- 配置通过 env 注入
- 日志输出到 stdout
- 服务支持热重载

## 交付建议

- README 中写清楚一键启动命令
- 每个服务提供 health endpoint
- 本地环境和生产环境配置分离
