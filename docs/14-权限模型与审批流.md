# 14 权限模型与审批流

## 目标

限制 Hermes 在边缘侧和云端侧能做什么，避免把一个强执行系统变成高风险工具。

## 权限维度

### 1. 按租户
不同组织隔离。

### 2. 按区域
某个边缘节点只管自己区域。

### 3. 按设备类型
主机、IoT、网络设备权限不同。

### 4. 按动作
- 读状态
- 重启
- 改配置
- 升级
- 回滚

### 5. 按风险等级
- 低风险：自动允许
- 中风险：策略允许或条件审批
- 高风险：人工审批

## 审批流建议

1. Hermes 生成任务草案
2. Policy Service 判断风险
3. 低风险自动通过
4. 中高风险进入审批队列
5. 审批通过后下发执行
6. 结果自动回写审计

## 审批信息

- 发起人
- 审批人
- 审批时间
- 任务摘要
- 变更范围
- 回滚方案

## 策略建议

- 默认拒绝高危动作
- 白名单优先于自然语言请求
- 对生产环境比测试环境更严格
- 大批量操作必须分批执行

## Hermes 的角色

Hermes 不负责最终授权决定，它只负责：
- 解释意图
- 提交任务建议
- 按策略执行已批准任务
