# 05 任务编排与执行模型

## 基本流程

1. 云端创建高层任务
2. Hermes 解析目标与约束
3. 拆解为 JobStep
4. 选择区域与设备
5. 调用适配器执行
6. 校验结果
7. 必要时回滚
8. 回传审计记录

## 任务类型

### 1. 巡检类
- 设备在线检查
- 服务健康检查
- 配置漂移检查
- 资源使用率采集

### 2. 变更类
- 参数调整
- 服务重启
- 配置分发
- 固件升级

### 3. 修复类
- 自动重启
- 回滚配置
- 替换证书
- 清理异常状态

### 4. 发现类
- 新设备发现
- 拓扑更新
- 版本识别
- 标签补全

## 执行策略

- 先 precheck
- 再执行
- 再 verify
- 最后审计

## 失败处理

- 局部失败不直接视为整体失败
- 关键步骤必须支持补偿
- 可回滚任务必须先生成回滚计划
- 重试要有上限和退避

## 适合 Hermes 的部分

Hermes 在这里的强项是：
- 把自然语言目标拆成步骤
- 根据上下文选择工具
- 处理执行中的动态变化
- 把经验写回记忆或 skills
