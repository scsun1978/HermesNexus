# 63 本地调试与模拟器

## 目标

让开发者不依赖真实站点，也能完成云边联调和设备适配开发。

## 模拟器类型

### 1. 节点模拟器
- 模拟边缘节点上线、心跳、任务回传
- 用于验证云端编排链路

### 2. 设备模拟器
- 模拟主机、IoT、网络设备
- 用于协议适配器开发

### 3. 告警模拟器
- 模拟监控事件触发
- 用于测试告警闭环

### 4. 网络故障模拟器
- 断网
- 延迟
- 丢包
- DNS 异常

## 调试目标

- 服务是否能启动
- 任务是否能派发
- 节点是否能注册
- 事件是否能回传
- 回滚是否能触发

## 本地联调顺序

1. 启动数据库和队列
2. 启动云端 API
3. 启动任务服务
4. 启动节点模拟器
5. 启动设备模拟器
6. 观察任务执行与事件回传

## 建议

- 所有模拟器都要可脚本化
- 失败场景要可重复
- 模拟器结果要写入测试报告
- 模拟器最好支持参数化运行
