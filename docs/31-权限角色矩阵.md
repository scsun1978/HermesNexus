# 31 权限角色矩阵

## 目标

把不同角色对不同对象、不同动作的权限明确下来。

## 角色

- 运维工程师
- 平台管理员
- 站点负责人
- 审批人
- 只读观察者
- 自动化 Agent

## 权限对象

- 资产
- 任务
- 策略
- 审计
- 记忆
- 告警
- 节点

## 权限动作

- 查看
- 创建
- 修改
- 删除
- 执行
- 审批
- 回滚
- 导出

## 矩阵原则

### 运维工程师
- 可查看资产和告警
- 可创建巡检和普通变更任务
- 中低风险动作可执行
- 高风险动作需审批

### 平台管理员
- 可管理策略、区域、节点、权限
- 可查看全量审计
- 可配置审批规则

### 站点负责人
- 可查看本站点整体状态
- 可审批部分站点动作
- 不一定能修改全局策略

### 只读观察者
- 只可查看
- 不可执行变更

### 自动化 Agent
- 仅可执行被授权的 API
- 不能越权发起高危动作

## 设计建议

- 采用 RBAC + 风险等级叠加
- 把“能做什么”与“是否必须审批”分开
- 生产环境和测试环境要不同策略
- 角色权限要支持按区域收敛
