# HermesNexus 风险分级规则文档

## 1. 风险分级概述

HermesNexus 风险分级系统用于评估云边设备管理平台中各类操作的风险等级，为权限控制和审批流程提供决策依据。

## 2. 风险等级定义

### 2.1 等级分类

| 风险等级 | 英文名称 | 数值 | 描述 | 颜色标识 |
|----------|----------|------|------|----------|
| **低风险** | LOW | 1 | 只读操作，不影响系统状态 | 🟢 绿色 |
| **中风险** | MEDIUM | 2 | 修改操作，可快速回滚 | 🟡 黄色 |
| **高风险** | HIGH | 3 | 关键变更，影响业务连续性 | 🔴 红色 |

### 2.2 风险等级特征

#### 低风险 (LOW)
- **特征**: 只读操作，不修改系统状态
- **影响范围**: 仅影响当前用户
- **回滚难度**: 无需回滚
- **审批需求**: 无需审批
- **示例操作**:
  - 查询设备状态
  - 查看任务列表
  - 读取日志文件
  - 查看配置信息

#### 中风险 (MEDIUM)
- **特征**: 修改操作，但影响可控
- **影响范围**: 可能影响其他用户
- **回滚难度**: 可以快速回滚
- **审批需求**: 通常无需审批，特殊情况除外
- **示例操作**:
  - 创建新设备
  - 修改设备描述
  - 更新任务优先级
  - 执行常规任务

#### 高风险 (HIGH)
- **特征**: 关键变更，影响业务连续性
- **影响范围**: 可能影响大量用户或业务
- **回滚难度**: 回滚复杂或耗时
- **审批需求**: 必须经过审批
- **示例操作**:
  - 删除设备
  - 注销节点
  - 重启服务
  - 修改网络配置
  - 权限变更

## 3. 风险评估算法

### 3.1 基础算法

```
最终风险 = max(
    基础风险 × 资源因子,
    高风险模式匹配结果
)
```

### 3.2 操作类型基础风险

| 操作类型 | 基础风险 | 说明 |
|----------|----------|------|
| **查询操作** | LOW | read, list, get, query, describe |
| **修改操作** | MEDIUM | create, update, modify, change, configure |
| **删除操作** | HIGH | delete, remove, unbind, deregister |
| **执行操作** | MEDIUM/HIGH | execute, start, stop, restart, deploy, rollback |
| **管理操作** | HIGH | approve, reject, grant, revoke, admin |

### 3.3 资源类型风险因子

| 资源类型 | 风险因子 | 说明 |
|----------|----------|------|
| **CONFIG** | 1.0 | 配置操作，保持原风险 |
| **ASSET** | 1.0 | 设备操作，保持原风险 |
| **TASK** | 1.0 | 任务操作，保持原风险 |
| **NODE** | 1.2 | 节点操作，风险提高20% |
| **USER** | 1.5 | 用户操作，风险提高50% |
| **TENANT** | 2.0 | 租户操作，风险提高100% |
| **REGION** | 1.5 | 区域操作，风险提高50% |
| **LOG** | 0.5 | 日志操作，风险降低50% |
| **AUDIT** | 0.3 | 审计操作，风险降低70% |

### 3.4 风险计算示例

**示例1: 查询设备状态**
```
基础风险 = LOW (1)
资源因子 = ASSET (1.0)
最终风险 = 1 × 1.0 = 1 → LOW
```

**示例2: 更新设备配置**
```
基础风险 = MEDIUM (2)
资源因子 = ASSET (1.0)
最终风险 = 2 × 1.0 = 2 → MEDIUM
```

**示例3: 删除用户**
```
基础风险 = HIGH (3)
资源因子 = USER (1.5)
最终风险 = 3 × 1.5 = 4.5 → 限制为3 → HIGH
```

**示例4: 修改租户配置**
```
基础风险 = MEDIUM (2)
资源因子 = TENANT (2.0)
最终风险 = 2 × 2.0 = 4 → 限制为3 → HIGH
```

## 4. 高风险模式识别

### 4.1 关键词匹配规则

系统通过以下关键词模式识别高风险操作：

| 模式 | 说明 | 示例 |
|------|------|------|
| `delete.*all` | 删除所有 | "delete all servers" |
| `shutdown` | 关闭系统 | "system shutdown" |
| `reboot.*all` | 重启所有 | "reboot all devices" |
| `wipe` | 擦除数据 | "wipe disk data" |
| `format` | 格式化 | "format disk" |
| `drop.*table` | 删除表 | "drop table users" |
| `truncate` | 清空表 | "truncate table" |
| `grant.*admin` | 授予管理员权限 | "grant admin privileges" |
| `disable.*security` | 禁用安全 | "disable firewall" |
| `bypass.*auth` | 绕过认证 | "bypass authentication" |

### 4.2 模式匹配示例

**正常删除操作**:
```python
context = {"description": "删除单个服务器"}
# 风险等级 = HIGH (正常删除风险)
```

**危险删除操作**:
```python
context = {"description": "delete all servers and shutdown system"}
# 风险等级 = HIGH (匹配高风险模式)
```

## 5. 需要确认的操作

以下操作类型需要用户额外确认：

| 操作类型 | 说明 | 确认提示 |
|----------|------|----------|
| **DELETE** | 删除操作 | "确认要删除吗？此操作不可撤销" |
| **DEREGISTER** | 注销操作 | "确认要注销吗？这将移除所有配置" |
| **RESTART** | 重启操作 | "确认要重启吗？服务将短暂中断" |
| **DEPLOY** | 部署操作 | "确认要部署吗？将替换现有版本" |
| **ROLLBACK** | 回滚操作 | "确认要回滚吗？将撤销最近变更" |
| **GRANT** | 授权操作 | "确认要授予权限吗？此操作影响安全" |
| **REVOKE** | 撤销操作 | "确认要撤销权限吗？用户可能失去访问" |

## 6. 需要审批的风险等级

| 风险等级 | 审批需求 | 审批流程 |
|----------|----------|----------|
| **LOW** | 无需审批 | 直接执行 |
| **MEDIUM** | 特殊情况需要审批 | 一般直接执行，敏感操作需审批 |
| **HIGH** | 必须审批 | 必须经过审批流程 |

### 6.1 审批流程

```
高风险操作 → 创建审批请求 → 等待审批 → 执行/拒绝
                ↓
            记录审计日志
```

## 7. 自定义风险规则

### 7.1 规则配置

系统支持自定义风险规则：

```python
custom_rules = [
    {
        "action": "update",
        "resource": "config",
        "conditions": {"config_type": "security"},
        "override_risk": "high"  # 强制为高风险
    },
    {
        "action": "delete",
        "resource": "asset",
        "conditions": {"asset_type": "test"},
        "risk_adjustment": -1  # 降低一个风险等级
    }
]
```

### 7.2 规则优先级

1. 高风险模式匹配 (最高优先级)
2. 自定义规则覆盖
3. 标准风险评估算法

## 8. 风险监控与告警

### 8.1 监控指标

- 高风险操作数量趋势
- 审批通过/拒绝率
- 风险等级分布
- 用户风险行为模式

### 8.2 告警规则

| 告警级别 | 触发条件 | 处理措施 |
|----------|----------|----------|
| **INFO** | 高风险操作增加 | 记录日志 |
| **WARNING** | 异常高风险模式 | 发送通知 |
| **CRITICAL** | 大量高风险操作 | 立即介入 |

## 9. 最佳实践

### 9.1 风险控制原则

1. **最小权限**: 只授予必要的权限
2. **职责分离**: 关键操作需要多人审批
3. **定期审查**: 定期审查风险规则配置
4. **持续监控**: 持续监控风险操作趋势

### 9.2 风险降低策略

1. **操作前确认**: 高风险操作前强制确认
2. **操作中监控**: 实时监控操作执行状态
3. **操作后验证**: 验证操作结果和系统状态
4. **异常恢复**: 准备快速回滚方案

## 10. 示例场景

### 10.1 场景1: 常规设备维护

```python
# 操作：查看设备状态
action = ActionType.READ
resource = ResourceType.ASSET
# 风险等级：LOW
# 审批需求：无需审批
# 确认需求：无需确认
```

### 10.2 场景2: 设备配置修改

```python
# 操作：修改设备描述
action = ActionType.UPDATE
resource = ResourceType.ASSET
# 风险等级：MEDIUM
# 审批需求：无需审批
# 确认需求：无需确认
```

### 10.3 场景3: 生产设备删除

```python
# 操作：删除生产设备
action = ActionType.DELETE
resource = ResourceType.ASSET
context = {"environment": "production"}
# 风险等级：HIGH
# 审批需求：需要审批
# 确认需求：需要确认
```

### 10.4 场景4: 租户配置变更

```python
# 操作：修改租户配置
action = ActionType.UPDATE
resource = ResourceType.TENANT
# 风险等级：HIGH (MEDIUM × 2.0)
# 审批需求：需要审批
# 确认需求：需要确认
```
