## 1. 身份主体分类

### 1.1 用户
**定义**: 操作HermesNexus系统的操作人员

**分类**:
- **管理员** - 拥有系统所有权限
- **操作员** - 可以创建和执行任务
- **只读用户** - 仅能查看系统状态

**认证方式**:
- API Key (长期凭证)
- JWT Token (会话凭证)

**权限范围**:
- 资产管理: 创建、查看、更新、删除
- 任务管理: 创建、查看、取消、查看结果
- 系统管理: 用户管理、节点管理、系统配置

### 1.2 边缘节点
**定义**: 执行系统任务的计算节点

**身份属性**:
- **节点ID**: 全局唯一标识符 (UUID或预分配ID)
- **节点名称**: 便于识别的显示名称
- **节点类型**: 物理机、虚拟机、容器
- **地理位置**: 数据中心、机房、机架
- **能力清单**: 支持的协议、并发任务数、系统资源

**认证方式**:
- 节点JWT Token (包含节点ID和权限)
- 节点证书 (mTLS，生产环境)

**权限范围**:
- 注册和心跳上报
- 获取分配给本节点的任务
- 上报任务执行结果
- 报告节点状态和错误

### 1.3 设备
**定义**: 被管理的目标资产

**身份属性**:
- **设备ID**: 全局唯一标识符
- **设备类型**: 服务器、网络设备、IoT设备
- **连接信息**: IP、端口、认证信息
- **所属节点**: 关联的管理节点

**认证方式**:
- 通过所属节点的身份间接认证
- 设备级别的访问凭证 (SSH密钥、密码)

**权限范围**:
- 被动执行，不具备主动权限
- 访问范围限定在所属节点的任务内

### 1.4 自动化Agent
**定义**: 系统内部自动化流程的身份

**分类**:
- **系统服务**: 定时任务、清理任务
- **审批流程**: 自动审批Agent
- **监控Agent**: 告警处理和自动恢复

**认证方式**:
- 内部服务Token
- 系统用户身份

**权限范围**:
- 限定在特定的自动化功能域
- 操作可追溯和可审计

## 2. 身份边界关系

### 2.1 用户 → 节点 → 设备 三层关系

```
┌─────────────┐
│   用户      │
│ (操作者)    │
└──────┬──────┘
       │ 创建任务
       ↓
┌─────────────┐
│   节点      │
│ (执行者)    │
└──────┬──────┘
       │ 管理
       ↓
┌─────────────┐
│   设备      │
│ (目标)      │
└─────────────┘
```

### 2.2 认证边界矩阵

| 主体 | 认证方式 | 权限来源 | 访问范围 |
|------|----------|----------|----------|
| 用户 | API Key/Token | 角色分配 | API端点 |
| 节点 | 节点Token/证书 | 注册时分配 | 云边通信 |
| 设备 | 节点代理认证 | 所属节点 | 执行任务 |
| Agent | 服务Token | 系统配置 | 特定功能域 |

## 3. 认证流程设计

### 3.1 用户认证流程

```
1. 用户登录 → 验证凭据 → 颁发JWT Token
2. API调用 → 携带Token → 中间件验证 → 允许/拒绝
3. Token过期 → 刷新Token → 继续使用
4. 用户登出 → 撤销Token → 清理会话
```

### 3.2 节点认证流程

```
1. 节点启动 → 生成节点ID和密钥对
2. 首次注册 → 提交节点信息 → 云端验证 → 颁发节点Token
3. 定期心跳 → 携带Token → 云端验证 → 更新状态
4. 任务执行 → 使用Token获取任务 → 上报结果
5. 节点下线 → Token失效 → 清理资源
```

### 3.3 设备访问流程

```
1. 节点获取任务 → 验证Token → 获取任务详情
2. 节点连接设备 → 使用设备凭证 → 建立连接
3. 执行任务命令 → 获取结果 → 返回云端
4. 连接关闭 → 清理资源 → 记录状态
```

## 4. 节点身份模型

### 4.1 节点身份数据模型

```python
class NodeIdentity(BaseModel):
    """节点身份模型"""
    # 基本身份信息
    node_id: str                    # 节点唯一ID
    node_name: str                  # 节点名称
    node_type: str                  # 节点类型
    
    # 认证信息
    auth_token: str                 # 当前认证Token
    token_expires_at: datetime       # Token过期时间
    public_key: str                 # 公钥(可选，用于mTLS)
    
    # 能力信息
    capabilities: Dict[str, Any]    # 能力清单
    max_concurrent_tasks: int       # 最大并发任务数
    
    # 状态信息
    status: NodeStatus              # 节点状态
    registered_at: datetime         # 注册时间
    last_heartbeat: datetime        # 最后心跳时间
    
    # 关联关系
    managed_devices: List[str]      # 管理的设备ID列表
    assigned_tasks: List[str]       # 分配的任务ID列表
```

### 4.2 节点生命周期状态

class NodeStatus(str, Enum):

## 5. 认证技术方案

### 5.1 节点Token方案 (JWT)

**Token结构**:
```json
{
  "header": {
    "alg": "RS256",
    "typ": "JWT"
  },
  "payload": {
    "node_id": "node-001",
    "node_name": "data-center-1-node-1",
    "node_type": "physical",
    "permissions": ["task.execute", "task.report", "heartbeat"],
    "issued_at": "2026-04-12T10:00:00Z",
    "expires_at": "2026-04-13T10:00:00Z"
  }
}
```

**安全特性**:
- 使用RS256非对称加密
- 云端使用私钥签名，节点使用公钥验证
- Token有效期24小时，可自动刷新
- Token吊销机制

### 5.2 云边通信认证集成

**中间件设计**:
```python
class NodeAuthMiddleware:
    """节点认证中间件"""
    
    async def authenticate_node(self, request):
        """验证节点身份"""
        # 1. 提取Token
        token = self.extract_token(request)
        
        # 2. 验证Token签名
        payload = self.verify_token(token)
        
        # 3. 检查权限
        self.check_permission(payload, request)
        
        # 4. 记录审计日志
        self.log_access(payload, request)
```

## 6. 安全边界定义

### 6.1 访问控制边界

**用户访问边界**:
- API端点访问控制
- 资源级别权限控制
- 操作级别权限控制

**节点访问边界**:
- 仅能访问分配给本节点的任务
- 仅能上报本节点相关的状态
- 无法访问其他节点的信息

**设备访问边界**:
- 通过所属节点的身份访问
- 执行限定在任务范围内
- 无法主动发起连接

### 6.2 数据隔离边界

**用户数据隔离**:
- 按租户/组织隔离
- 按角色权限隔离
- 审计日志完整记录

**节点数据隔离**:
- 节点间任务数据隔离
- 节点状态数据隔离
- 设备连接信息隔离
