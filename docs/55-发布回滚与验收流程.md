# 55 发布回滚与验收流程

## 目标

让 Hermes 的每次发布都可控、可回退、可验证。

## 发布前检查

- 代码已合并
- 测试已通过
- 配置已更新
- 迁移脚本已准备
- 回滚方案已准备

## 发布流程

1. 生成发布版本
2. 部署到测试环境
3. 验证关键链路
4. 灰度到部分区域
5. 观察指标
6. 全量发布

## 回滚条件

- 节点注册失败率高
- 任务执行失败率高
- 关键 API 异常
- 告警无法闭环
- 审计异常

## 回滚动作

- 切回旧版本
- 恢复配置
- 恢复数据库备份
- 重新发布旧镜像

## 验收流程

- 检查节点在线
- 检查任务可执行
- 检查事件可追踪
- 检查告警可联动
- 检查审计可查询

## 发布后观察

- 24 小时内重点看错误率
- 监控关键指标趋势
- 记录问题和修复动作

## 建议

- 所有发布必须有回滚按钮
- 灰度阶段必须有明确范围
- 验收结果必须形成记录
