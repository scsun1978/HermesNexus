# 40 云边协议细节

## 目标

定义云端控制平面和边缘 Hermes 节点之间具体怎么对话。

## 协议目标

- 可靠
- 可追踪
- 可重放
- 可签名
- 支持断线恢复

## 消息类型

### 云到边缘
- task.assign
- policy.update
- config.push
- credential.rotate
- node.command

### 边缘到云
- node.heartbeat
- task.accepted
- task.progress
- task.finished
- task.failed
- event.report

## 推荐协议内容

每条消息应该包含：
- message_id
- trace_id
- sender_id
- receiver_id
- region_id
- type
- timestamp
- payload
- signature

## 可靠性要求

- 幂等处理
- 重试机制
- ACK 确认
- 超时控制
- 顺序保证（同一任务内）

## 安全要求

- 消息签名
- 证书校验
- 过期时间
- 防重放

## 断线处理

- 边缘节点本地缓存消息
- 恢复后补传
- 云端基于 message_id 去重

## 建议实现

- 早期：HTTP + 签名 + ACK
- 中期：gRPC streaming
- 后期：消息队列 + 事件流

## Hermes 的参与方式

Hermes 可以在协议层负责：
- 生成任务消息
- 理解回包
- 提供失败原因解释
- 根据状态自动补发或重试
