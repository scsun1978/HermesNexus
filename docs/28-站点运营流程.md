# 28 站点运营流程

## 目标

把 Hermes 在单个区域/站点里的日常运营流程标准化，方便复制到更多现场。

## 日常流程

### 1. 设备接入
- 发现设备
- 验证协议
- 绑定区域
- 写入资产库

### 2. 日常巡检
- 运行健康检查
- 检查关键服务
- 检查连接状态
- 输出日报

### 3. 变更管理
- 提交变更计划
- 审批
- 分批执行
- 验证
- 记录审计

### 4. 故障处理
- 接收告警
- 收集现场数据
- 匹配历史经验
- 采取修复动作
- 回写结果

### 5. 经验沉淀
- 把处理过程写入记忆
- 将高频流程抽象成 skill
- 归纳成站点级规则

## 站点健康指标

- 设备在线率
- 告警处理时长
- 任务成功率
- 回滚率
- 变更失败率
- 经验复用率

## 站点运营建议

- 每个站点有自己的边缘节点
- 每个站点维护自己的经验摘要
- 重大操作必须有审批和回滚
- 站点级知识应可迁移到新站点

## Hermes 的价值

Hermes 让站点运营从“人肉操作”变成“可编排、可验证、可学习的系统化流程”。
