# 39 认证与密钥管理方案

## 目标

确保云端、边缘节点、设备和用户之间的身份可信、权限明确、密钥可轮换。

## 身份主体

- 云端服务
- 边缘 Hermes 节点
- 设备
- 运维人员
- 自动化 Agent

## 认证方式

### 1. 节点认证
- mTLS 证书
- 节点唯一 ID
- 短期 token

### 2. 用户认证
- SSO / OAuth / 企业账号
- MFA（可选）
- 角色绑定

### 3. 设备认证
- SSH key
- API token
- 设备证书
- 厂商专用凭证

## 密钥管理原则

- 云端集中签发
- 边缘只持短期凭证
- 证书可轮换
- 证书可吊销
- 敏感信息不进日志

## 建议机制

### 1. Vault / KMS
- 存储密钥
- 动态签发短期 token
- 审计访问记录

### 2. Secret 分层
- 全局密钥
- 区域密钥
- 设备级密钥

### 3. 访问控制
- 最小权限
- 按区域隔离
- 按动作隔离

## 安全流程

1. 节点注册
2. 云端签发证书
3. 节点拉取任务
4. 任务执行时只使用短期凭证
5. 到期自动轮换
6. 异常时吊销

## 设计建议

- 不要把长期密钥散落在边缘机器上
- 证书失效要有恢复流程
- 高危动作审批后再换取执行凭证
