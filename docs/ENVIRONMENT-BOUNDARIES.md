## 📋 依赖管理策略

### Hermes依赖处理原则

**核心原则**: Hermes作为"受控依赖"而非"系统级前置条件"

**具体策略**:
1. **版本固定**: 在requirements.txt中固定Hermes相关依赖版本
2. **项目隔离**: 使用项目专用虚拟环境，不依赖系统全局Hermes
3. **配置分离**: 使用独立的HERMES_HOME配置目录
4. **按需使用**: 仅在边缘节点运行时使用Hermes功能

**依赖版本固定**:
```
# 核心框架
fastapi==0.104.1
uvicorn[standard]==0.24.0
pydantic==2.5.0

# SSH和设备管理
paramiko==3.3.1
asyncio==3.11.0

# 数据持久化
aiosqlite==0.19.0

# 监控和日志
psutil==5.9.6
```

## 🔄 环境切换指南

### 本机开发环境设置
```bash
# 1. 进入项目目录
cd /Users/shengchun.sun/Library/CloudStorage/OneDrive-个人/MyCloud/Code/HermesNexus

# 2. 激活虚拟环境

# 3. 使用开发配置

# 4. 启动开发服务器
```

## 🚨 环境边界规则

### ✅ DO (允许事项)

**本机环境**:
- ✅ 使用项目虚拟环境进行开发
- ✅ 编写和运行单元测试
- ✅ 使用内存数据库快速迭代
- ✅ 调试配置文件和脚本

**开发服务器**:
- ✅ 部署控制平面和边缘节点
- ✅ 使用SQLite持久化数据库
- ✅ 运行集成测试和E2E验证
- ✅ 监控服务健康状态
- ✅ 收集和分析日志

### ❌ DON'T (禁止事项)

**本机环境**:
- ❌ 不直接运行生产级边缘节点
- ❌ 不使用系统全局Hermes环境
- ❌ 不进行长期稳定性测试
- ❌ 不运行生产级数据库

**开发服务器**:
- ❌ 不进行日常代码开发
- ❌ 不频繁修改部署配置
- ❌ 不承担最终生产环境职责
- ❌ 不存储敏感生产数据
