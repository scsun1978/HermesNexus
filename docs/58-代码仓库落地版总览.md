# 58 代码仓库落地版总览

## 目标

把 Hermes 分布式边缘设备管理系统从概念设计落到一个真正可开发的代码仓库方案。

## 落地原则

- 先 monorepo，后按边界拆分
- 先共享协议，后服务实现
- 先最小可运行，后多区域扩展
- 先本地可调试，后云边发布

## 仓库应该包含什么

- 云端服务
- 边缘节点运行时
- 协议与共享模型
- 设备驱动与适配器
- 控制台前端
- 部署脚本
- 测试与模拟器
- 文档与示例
