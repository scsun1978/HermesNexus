# 06 记忆与知识体系

## 分层设计

### 1. 短期记忆
- 当前会话上下文
- 当次任务状态
- 临时约束
- 适合处理“刚发生的事”

### 2. 内置长期记忆
- MEMORY.md：环境事实、经验、 conventions
- USER.md：用户偏好、沟通方式、角色信息

### 3. 外部记忆 provider
用于更深层的长期知识：
- Honcho：用户画像与多 agent 关系
- OpenViking：分层知识结构
- Mem0：自动事实抽取
- Hindsight：知识图谱和关系推理
- Holographic：本地 SQLite 检索
- RetainDB / ByteRover / Supermemory

## MVP 建议

- MVP 阶段不强制接入 OpenViking。
- 先把记忆能力抽象成 provider 接口，底层可以先用本地 SQLite / PostgreSQL / 简单文件存储实现。
- OpenViking 适合作为第二阶段的云端共同记忆体或知识分层引擎接入。
- 如果 MVP 必须体现记忆能力，只保留“读 / 写 / 检索”的最小接口，不做复杂知识编排和自动分层。

### 4. 云端知识库
- 全局设备知识
- 故障案例库
- 变更记录
- 设备画像
- 站点画像

## 建议策略

- 边缘节点优先维护站点记忆
- 云端维护全局知识与策略
- 复杂故障案例沉淀成技能 + 知识条目
- 设备类经验尽量结构化，避免只留日志

## 记忆写入原则

- 简短
- 可复用
- 可检索
- 可验证
- 与具体设备/站点/流程强相关

## 一个好记忆条目应该包含

- 背景
- 现象
- 原因
- 处理方式
- 可复用规则
